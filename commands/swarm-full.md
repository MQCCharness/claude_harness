# 满编蜂群（五段 R1–R4b）— Claude Code 适配版

请在本仓库按 **满编蜂群** 执行本条及后续追问。

## 满编蜂群定义

**R1、R2、R3、R4a、R4b 各至少一次 Agent 子代理**（**R4 计两次**：R4a 清单 + R4b 核验），遵守蜂群编队时序闸门。

## 强制时序

```
闸门 0（EXEC_CFG_GATE）：执行配置关门
    ↓
R1 + R2 并行派发 → 两者均关门
    ↓
闸门 1 满足 → R3 启动（输入须含 R1/R2 摘要）
    ↓
R4a 与 R3 并行（仅契约级，不喂实现细节）
    ↓
R3 关门 + 统帅实测留痕
    ↓
R4b（对照 R4a 冻结表 + R3 证据做事实核对）
    ↓
统帅按 R4a 表亲测 → PASS / COND_PASS / FAIL
```

## 角色固化（派发前）

按 `CLAUDE.md §六` 与 `AGENTS.md` 完成：
- **(α)** 各角色 Task 定稿（首行锚点 `CHAIN=<ID> ROUND=<N> ROLE=R1|R2|R3|R4a|R4b`）
- **(β)** 更新 `.claude/agents/runs/<CHAIN-ID>/swarm-*.md`（含 `## 本链绑定` + `### 最近派发 Task 摘要`）
- **(γ)** 派发 Agent 子代理（prompt 含 `ROLE_DEF=@路径`，且 EXEC_CFG_GATE 已关门或豁免）

## 输入要求

- **R1**：目标一句 + 硬约束 + 假设风险面（禁止预灌实现细节）
- **R2**：待核实问题列表 + 关键词 + 已知官方入口
- **R3**：R1→R3 + R2→R3 条目化摘要 + 相关路径/模块边界
- **R4a**：契约级输入（REQ、冻结目标、环境矩阵），**禁止**纳入 R3 实现细节
- **R4b**：R4a 冻结验收表 + R3 关门摘要 + 可运行证据

## 验收裁决

- **PASS**：P0 全部实测通过
- **COND_PASS**：P0 通过但存在未自动化的 P1 或已文档化的剩余风险
- **FAIL**：任一 P0 未通过 → 修复后重跑门禁，不得收束
