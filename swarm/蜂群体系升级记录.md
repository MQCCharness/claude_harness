## 2026-03-22 — 任务链隔离、留痕三档、双轨验收与脚本工具链

**升级特性**

- **任务链（CHAIN）与父会话隔离**：不同冻结目标/交付物类型推荐**新父会话**；同会话多链须 **CHAIN + 轮次表** 留痕，防子代理目录「一桶杂烩」。
- **Task 首行锚点**：满编或显式 R1–R4 时 **第一行强制** `CHAIN` / `ROUND` / `ROLE` / `PARENT_REF`；**禁止**用子代理文件 mtime 冒充因果序。
- **留痕三档**：叙事稿 / 结构化全记录 / 字节镜像；与 **`16` 的 L0–L3 触发分层**明确**不同名**；冲突时用 **`OMIT_REASON`** 封闭枚举（技能 `12`/`14`/`08` 互指）。
- **统帅汇流骨架**：冻结事实 → 转化表 → 与用户旧结论矛盾 → 诚实边界 → 待决与指针（含 CHAIN/ROUND）。
- **R4 分轨**：**TC-B-**（业务/方案黑盒）与 **TC-E-**（工程/构建）分列 P0 最小集合；端到端集成缺陷须明确主责轨或 COND_PASS 条件。
- **工具链**：可重复操作**入仓脚本**；**禁止**关键路径依赖未验证的**单行 PowerShell 拼接**。
- **技能 `08`**：从纯历史标题扩展为「会话留痕与导出」专篇，并指针 `_export_sales_swarm_full_md.py` 示例。

**好处举例**

1. **场景**：销售方案与导出器迭代共用一会话，子代理文件数暴涨、复盘对不上轮次。CHAIN + 新开会话 + 轮次表让「哪条链产出了什么」可 grep。
2. **场景**：用户既要「自然语言全记录」又要审计。三档 + OMIT 把**人读**与**字节对账**拆清，减少「同一文件既要无省略又要无代码」的需求打架。
3. **场景**：方案写得漂亮但 CI 没跑。TC-B / TC-E 分轨提醒统帅**两轨都要有关闭条件**，降低字面上双过、实际上漏端到端的概率。

**关联路径**：`CLAUDE.md §二`，`.claude/skills/hajimi-dog/03_AI行为准则/08_当前对话记录导出.md`，`12` / `14` / `16`，`00_全局索引/SKILL.md`，`./_export_sales_swarm_full_md.py`，`.claude/swarm/RECENT-DECISIONS.md`
