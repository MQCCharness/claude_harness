# Claude 环境专用：Agent 升级提示词（蜂群 + 军团 · 2026-03-27）

**适用**：Anthropic **Claude Code**、带 **项目/仓库文件访问** 的 Claude Agent、或用户在 Claude 侧挂载本仓库后让 Agent 改文件对齐治理栈。  
**与通用版关系**：语义与同主题 **「异地同步」** PROMPT（文件名含 `PROMPT-异地同步-蜂群与军团`）一致；本文件补充 **Claude 侧工具名、会话习惯与 Project Instructions** 注意点。

---

## 路径约定（跨环境）

- **不得假定**规则、技能、落盘目录一定在 `.claude/rules`、`.claude/skills/hajimi-dog` 等固定路径；**另一台机器或另一套工具链**可能把同名文件放在不同父目录。  
- **统一做法**：先在**当前可操作的项目/仓库根**下，按 **§3 文件名表** **搜索定位**得到实际路径，再 Read/Edit；验收命令里的路径一律用**你刚解析出的路径**代入。  
- **工作目录**：终端命令在「项目根」执行；项目根 = 用户指定或含治理文件的仓库根，**不绑定盘符与文件夹命名**。

---

## 0. Claude 环境注意（Agent 执行前必读）

1. **规则加载**：部分 IDE 会自动读规则文件；Claude **未必**自动注入。若仅用 Claude 操作仓库，可在 **Project Instructions / CLAUDE.md（若存在）** 中写明：会话前须 **搜索并 Read** 定位到的 **蜂群 / 军团** 主规则文件（见 §3 文件名）。**本升级任务**要求 Agent **直接编辑磁盘上的这些文件**，不依赖隐式加载。  
2. **子任务命名**：蜂群/军团条文中的 **Task、Sub-agent、R4a/R4b** 等，在 Claude Code 中对应 **Task 工具 / 子 Agent**（若产品提供）；若无，按规则中的 **诚实边界** 用**分节扮演**等效，但**不得**用单段混写冒充多角色。  
3. **终端**：验收用 **Bash**；若无 `rg`，可用 `grep -R` 或按文件名搜索后对单文件 `grep`。  
4. **输出语言**：若用户要求中文，终稿对用户说明用中文；**文件内容**保持与仓库现有语言风格一致。

## 1. 人读摘要（与通用 PROMPT 相同，便于单文件携带）

- **蜂群**：编队时序闸门 **R1+R2→R3**；**R4a**∥R3（仅契约级）；统帅实测；**R4b** 在 R3 关门+证据后；写码回合 **R3+R4a+R4b**；满编 **R1、R2、R3、R4a、R4b**；**PASS/COND_PASS/FAIL** 语义不变。  
- **军团**：**COND_PASS** 与蜂群对齐 + **COND_ID** 留痕；叶 OP Gate **FAIL 不放行**；回声 **PASS/BLOCK/COND_PASS** 与 **TC-ID / ACCEPT**、**R4a 表**互指；**收敛 + 5 轮熔断**（叶 OP / 军团）。  
- 详细条目不重复展开时，Agent 以 **§2 长提示词** 为执行清单。

## 2. 可复制长提示词（粘贴到 Claude 对话 / 项目说明）

下列整段可复制给 **Claude 环境中的 Agent**（单条用户消息即可）。

```text
【环境】你在 Claude（含 Claude Code 或等效可读写仓库的 Agent）中操作已包含 HajimiDog 治理文件的项目。

【路径解析 — 必须先做】勿假定规则/技能在固定目录（如某 IDE 的默认 .cursor 结构）。请在项目根下按**文件名**搜索并记录**真实路径**，后续凡写「蜂群规则文件」「军团规则文件」「技能12」「技能20」「linter」「裁决简报」均指你解析到的路径：
- CLAUDE.md §二 → 蜂群主规则
- CLAUDE.md §三 → 军团主规则
- 12_蜂群规则与技能体系维护指南.md → 技能12
- 20_军团模式回声校验与规模裁量.md → 技能20
- skills_linter.py → 技能体系 linter（若存在）
- RECENT-DECISIONS.md → 裁决简报（通常与 swarm 落盘同树；若有多份，选与本治理栈配套的那份）

【角色】你是统帅（+1）。任务：完成「蜂群 4+1 + 军团 L3」治理栈的 **2026-03-27 升级对齐**。开始前完成路径解析；用文件工具阅读相关文件、用搜索确认关键词；再最小必要地编辑；最后在项目根对解析出的路径执行验收。

【冻结语义 A — 蜂群 L2】
1. 蜂群规则文件须含章节「蜂群编队启动与时序闸门」：R1+R2 关门前不得派发 R3；R4a 可与 R3 并行且子任务仅限契约级、不得纳入 R3 成果；统帅按 R4a 表实测；R4b 仅在 R3 关门且证据回灌后派发。
2. 写码回合最低触达须写清 **R3 + R4a + R4b**（禁止用含糊的「仅 R3+R4」替代时序）。
3. 满编技术最小集：**R1、R2、R3、R4a、R4b** 各至少一次子任务（平台可用时）；Task 锚点 ROLE 支持 R4a|R4b。
4. R4 门禁三段式：R4a 冻结表 → 统帅实测 → R4b 核对；PASS/COND_PASS/FAIL 与蜂群规则文件一致。
5. 技能12与蜂群主规则一致，无「R4 先读实现再写清单」的旧误读主导叙述。

【冻结语义 B — 军团 L3】
1. 军团规则文件：不收窄蜂群底线；写码用语 **R3+R4a+R4b**。
2. Phase Gate：PASS/BLOCK/COND_PASS；CONDITIONAL_PASS 与 COND_PASS 同义；COND_PASS 须 COND_ID、条件、负责方、时限、关联 TC-ID；语义不弱于蜂群。
3. 叶 OP Gate：仅 PASS 或合法 COND_PASS 可放行；FAIL 不得放行。
4. 双层门禁：蜂群 FAIL 不得被军团全局 PASS 覆盖；蜂群合法 COND_PASS 不得被军团误杀。
5. 回声：PASS/BLOCK/COND_PASS；与 TC-B-/TC-E- 互指；主观项须 ACCEPT+TC-ID 方 A 级；与叶 OP R4a 表互指。
6. 收敛与熔断：叶 OP 同一目标 5 次 R3→R4b 不收敛须强制 R1+R2 重入；军团同一 CHAIN 回声 5 次仍 BLOCK 须强制 R1 型+R2 型子任务或改契约。
7. 技能20含 §0.2a–0.2c、模板 TC-ID/COND_ID、§3.3 P0 为 R3+R4a+R4b、§4 与蜂群衔接。

【执行步骤】
1. 在蜂群/军团规则文件与技能12、技能20中搜索：R4a、R4b、蜂群编队启动与时序闸门、收敛、轮次熔断、COND_PASS；列出缺口。
2. 若有已升级参考副本：优先整文件覆盖；否则按冻结语义逐段补写。
3. 若存在 linter：`python <解析到的 skills_linter.py 路径>`，期望退出码 0。
4. 在解析到的裁决简报文末追加一节：日期 + 「Claude 环境同步 / 蜂群 R4a·R4b + 军团 COND_PASS·收敛熔断」+ **实际编辑过的路径** + 已运行命令摘要。
5. 向用户回报：已改文件列表、未改的演示链/旧文案（P1）、是否需在 CLAUDE.md 增加「会话须先搜索并读蜂群/军团规则」一句。

【Claude 特有条目】
- 不要假设仅适用于某一 IDE 的目录结构；以「搜索得到的文件路径」为唯一权威。
- 若无法创建子 Agent：在交付说明中标注「诚实边界」与等效分节；仍须完成文件升级与 linter（若有）。

【禁止】
- 口头 PASS 替代 P0 实测；军团 PASS 覆盖蜂群 FAIL；R4a 子任务默认塞满实现细节（除非用户明示白盒）。
```
