# 记忆接力 · 通用宿主提示词（Portable）

> **⚠️ 效力范围（必读）**  
> 本文件可**整份复制**到任意项目或外机；**单独存在**不等于自动获得你团队内部的 Sub-agent 闸门、测试门禁或阶段 Gate——那些须在**宿主规则 / CI** 中另行定义。  
> **IDE Summarize / 总结对话**：须用户在 **IDE 内**操作；本机制**不**承诺用自然语言代替点击「总结」。  
> **权威**：外存与聊天矛盾时，以 **Read 到的落盘内容**为准，并回写指针或摘要。

---

## 0. 在解决什么问题

| 侧 | 职责 |
|----|------|
| **外置工作记忆（写）** | 把仍有效的**目标、约束、进度、拍板、验收状态**结构化写到仓库（或约定目录），减轻主会话 token 压力。 |
| **记忆接力（读）** | 在**新会话、上下文压缩后、或用户口令触发**时，主会话（主编排助手）按**固定顺序**读回这些文件，再派工或下结论。 |

二者配合才能实现「窗口干净、任务可续」。

### 渐进披露（与技能 22 对齐）

主帅按窗口冷热与是否压缩，**递进**读 L1 对话要点 → L2 `CHAIN-STATE` → L3 磁盘全文；**勿与**技能 16 的「L0–L3 触发分层」混为一谈。Summarize 后须遵守与口令记忆接力**等价的 Read 序**（至少 HANDOFF+§A），且**不替代** §A/HANDOFF 的维护。细则：`.claude/skills/hajimi-dog/03_AI行为准则/22_主帅上下文生命周期管理.md` §12.4。  
外机单文件泛化版：`.claude/swarm/PORTING-PROMPT-PROGRESSIVE-MEMORY-ASK.zh-CN.md` 或同文 **`.claude/swarm/渐进式记忆.md`**（含 **执行大环境**须落盘：仓库根、路径、ENV 名与指针、容器约定等，**禁止仅聊天**；两文件须同步维护）。

## 1. 宿主最小配置（复制本表到 README 或规则，并填路径）

下列占位符**由宿主项目自行定义**；未定义前，助手不得猜测路径冒充 **A 级**事实。

| 占位符 | 含义 | 填表示例（仅示意） |
|--------|------|-------------------|
| `{STATE_ROOT}` | 治理/蜂群根目录 | `.claude/swarm` |
| `{CHAIN_ID}` | 当前任务链标识 | `PROJ-FEATURE-001` |
| `{CHAIN_DIR}` | **蜂群式**单链目录 | `{STATE_ROOT}/chains/{CHAIN_ID}` |
| `{LEGION_CHAIN_DIR}` | **军团式**单链目录（可选） | `{STATE_ROOT}/legion/chains/{CHAIN_ID}` |
| `{FILE_HANDOFF}` | 会话切割稿（可选多份） | `{CHAIN_DIR}/HANDOFF-R{N}.md` |
| `{FILE_STATE}` | 链状态主文件 | `{CHAIN_DIR}/CHAIN-STATE.md` |
| `{SECTION_MINIMAL}` | 主文件内**最小可读摘要区**（如前 20 行、或名为 §A 的小节） | 由模板约定 |
| `{FILE_DECISIONS}` | 跨链/跨日简报（可选） | `{STATE_ROOT}/RECENT-DECISIONS.md` |
| `{FILE_L1_MIRROR}` | L1 草稿镜像（可选） | `{CHAIN_DIR}/L1-MIRROR.md` |
| `{DIR_TASK_REFLUX}` | 子任务回流摘要目录（可选） | `{CHAIN_DIR}/task-reflux/` |
| `{FILE_MISSION}` | 军令状/需求穿透（可选） | `{LEGION_CHAIN_DIR}/MISSION-BRIEF.md` |

**命名自由**：`CHAIN-STATE.md`、`HANDOFF` 等仅为**推荐约定**；宿主可改名，但须在配置表中**写死映射**，并在口令触发时仍能解析。

## 2. 触发口令（可增删；建议写入宿主规则并显式列举）

以下**任一**可视为用户要求执行 **记忆接力**（可用英文或其它语言做本地化，但须**书面固定**）：

- `记忆接力` · `记接力` · `摘要后接力` · `外存接力` · `磁盘接力`
- 英文示例：`memory relay` · `reload disk state` · `read handoff first`

**触发后的纪律**：主编排助手在本轮答复中，在**继续派下游任务、宣布验收结论、或开始实质性改代码/改配置**之前，须先完成 **§3 Read 序**。

## 3. 强制 Read 序（算法 · 与具体文件名解耦）

以下按**逻辑顺序**描述；实际路径用 **§1 配置表**替换。

1. **`{FILE_HANDOFF}`（若存在）**  
   - 多份时默认取**序号最大**的 `R{N}`，除非用户指定 `N`。  
   - 若宿主同时有「蜂群链目录」与「军团链目录」，先读用户声明的类型；未声明则**追问一句**或从 HANDOFF 抬头元数据推断（推断标 **B**）。

2. **`{FILE_STATE}` 的 `{SECTION_MINIMAL}`**  
   - 目标句、进行中的 P0、当前裁决、干系人/用途等**最小集合**。  
   - **若文件不存在**：答复**首段**声明「本链无状态主文件」，用 `{FILE_DECISIONS}` + 用户补述**降级**；**禁止**编造冻结事实。

3. **（可选 · 若存在）** `{FILE_L1_MIRROR}` 最近一轮块；`{DIR_TASK_REFLUX}` 下与当前轮次相关的摘要。

4. **（按需）** `{FILE_DECISIONS}` 中与 `{CHAIN_ID}` 或当前主题相关的**最新**条目。

5. **（若启用军团/军令状）** `{FILE_MISSION}` 及其中标记为**不可仅摘要替代**的原文节（若有此类约定）。

**冲突处理**：落盘与聊天不一致 → **以落盘为准**，并在对话中说明已对齐或待用户推翻。

**CHAIN_ID 缺失**：须**一句追问**；或从 HANDOFF / 状态文件抬头 / 用户上一句指针推断（标 **B**）。**禁止**无链标识时跨链盲读。

## 3A. 接续任务时的信息检索优先级（推荐 · 军团/蜂群）

**适用**：已冻结 **CHAIN_ID**、正在**接续**同一条蜂群链或军团链（非从零踩仓库）；主帅需要「背景、约束、上轮拍板、验收状态、部门产出指针」等信息时。

**目的**：减少**重复扫全仓**、减少与**已冻结形状**不一致的「现场发挥」；与 OpenClaw「先笔记后乱搜」同构，但本体系的真源是 **`.claude/swarm` 下链目录 + 军令状**。

**建议顺序（先读后扩）**

1. **`{LEGION_CHAIN_DIR}`（若存在本链军团目录）**  
   - 优先：`MISSION-BRIEF.md`、`ARCHITECTURE.md`、`SCHEDULE.md`、`README.md`、`ECHO-CHECK.md`、`PHASE-GATE-*.md`  
   - 其次：`artifacts/`、`rollups/`（若存在）等**本链产出**  
2. **同 ID 的蜂群链 `{CHAIN_DIR}`**（可与军团链并存）  
   - `HANDOFF-R{N}.md`（取最大 N 或用户指定）→ `CHAIN-STATE.md`（至少 §A，按需 §B–§G）  
3. **`{FILE_DECISIONS}`**  
   - 用 **CHAIN_ID / 主题** 过滤相关条目，**禁止**把无关链的简报当本链 A 级事实  
4. **（按需）** `{FILE_L1_MIRROR}`、`{DIR_TASK_REFLUX}`  
5. **再扩大到工程源码树**（`SemanticSearch` / `grep` / `Read` 业务代码）

**硬边界（例外 · 仍须下钻工程）**

- **运行时事实**（能否编译、测试红绿、实际 API 行为）以**代码 + 命令输出**为准；落盘若与之一致性未验证，落盘**不**自动覆盖可执行事实。  
- **BUG 定位、依赖版本、未写入军令状的实现细节**允许直接读代码；但若与**军令状②③④**冲突，须**显式报矛盾**（形状保持），不得静默以代码覆盖冻结目标。

**与 §3 的关系**：用户触发 **记忆接力** 时，仍须先完成 **§3 强制 Read 序**；§3A 是**接续轮次中「额外要找信息时」**的**检索优先序**，不替代 §3。

## 4. 可复制提示词块（宿主无关 · 粘贴即用）

### 4.1 给最终用户（贴在需求末尾）

```text
【记忆接力】CHAIN_ID=<填写>。
请按宿主约定的顺序读取外置状态：HANDOFF（若有）→ 状态主文件最小摘要区 →（若有）L1 镜像 / 任务回流 →（按需）裁决简报 →（若军团）军令状相关节。读完再派工或下验收结论。
```

### 4.2 给主编排助手 / Agent（冷启动或口令已触发）

```text
用户已触发记忆接力。本轮第一件事：按项目配置的 STATE_ROOT / CHAIN_DIR 完成读盘序；不得仅凭聊天梗概继续派单或宣布通过。
顺序：HANDOFF（若存在）→ 状态主文件最小区 → 可选 L1 镜像与 task-reflux → 按需 RECENT-DECISIONS 类简报 → 若有军令状则读相关节。
若无状态主文件，首段声明并降级；禁止虚构已冻结条款。IDE Summarize 不能替代上述 Read。
```

### 4.3 极简宿主（无自定义规则文件时）

```text
当用户说「记忆接力」或同义口令时：先问 CHAIN_ID 与状态文件路径；然后 Read 用户给出的 HANDOFF（若有）和「链状态」文件的前 20 行或约定小节；再回答。禁止未读盘即声称已对齐进度。
```
