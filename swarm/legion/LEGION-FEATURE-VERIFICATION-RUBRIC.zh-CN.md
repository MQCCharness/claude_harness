## 1. 检验对象与证据包（评分前置）

### 1.1 检验对象（须冻结）

| 字段 | 说明 |
|------|------|
| **CHAIN** | 任务链 ID；与 Task 首行锚点、轮次表一致 |
| **评分窗口** | 起止：消息范围 / 时间 / 或 `ROUND` 区间（须写明） |
| **链边界** | 单主题 / 多主题切分规则；未冻结则规模类（S/M/L）条目允许标 **N/A** 且不参与百分化分母（见 §6） |

### 1.2 证据包（最低集合 · P0）

检验团队应要求被评方提供可复核材料（脱敏后），**至少**能覆盖下列类型中的与本轮相关的子集：

| # | 证据类型 | 用途 |
|---|----------|------|
| E1 | 主对话摘录或导出（叙事/结构化档均可） | 激活、降级、用户确认、豁免声明 |
| E2 | 军令状（`MISSION-BRIEF` 或等价正文） | 五要素、① 原话是否改写 |
| E3 | 冻结三件套指针或节选：`ARCHITECTURE.md`、`MISSION-BRIEF.md`、`SCHEDULE.md` | 架构/DAG/I/O 冻结 |
| E4 | DAG + EP 表 + I/O 契约表（或 SCHEDULE 内嵌） | 无环、消费者驱动、decisions_log |
| E5 | Task 首行锚点样本（各 L3 节点 ≥1，叶 OP→蜂群衔接 ≥1） | `LEVEL`/`TIER`/`CHAIN`/`PARENT_REF` 等 |
| E6 | Phase Gate 与/或回声校验记录（技能 20 模板） | 双层门禁与终态对账 |
| E7 | 蜂群层：R4 裁决（PASS/COND_PASS/FAIL）及统帅实测留痕（写码回合） | L2 不得被 L3「盖住」 |
| E8 | `legion/skill-candidates/` 与 `INDEX.md`、`WAVE-CLOSE` F 块（若声称落实技能 21） | 对账与 §6 用户提问留痕 |
| E9 | 版本锚：`commit` / 构建 ID / 文档版本号（与评分所用 rubric 版本对齐） | 防版本脱钩 |

**缺 E2+E3+E5 中任一类**：除非文档声明 **N/A 理由**，否则军团核心特性（F5/F6）**不得记 3–4 分**。

### 1.3 证据等级（A / B / C）

与蜂群规则一致：**无「步骤 + 机器可读结果（退出码/等价物）+ 环境摘要」三者齐全，不得记 A 级**；无 A 时该细项**上限**见 §3.3。

## 2. 专业检验「军团」编组（任务型）

以下为**元架构**：检验工作可按多部门并行；**不强制**为每次检阅启动真实 Sub-agent，但须在汇总表声明「执行形态」（满编 Task / 抽检 / 统帅单上下文分节）。

| 部门 | 职责 | 主要覆盖特性 |
|------|------|----------------|
| **规范对齐组** | 对照 `CLAUDE.md §三` + 技能 18–21，查禁止项与模板字段 | F1–F4、F10–F11 |
| **穿透与军令组** | 五要素、②③⑤、[`PENETRATING`] 全链、追加区元数据、冲突上报 | F5 |
| **调度与契约组** | DAG↔I/O、EP、Gate 后裁剪、发包/代发留痕 | F6 |
| **汇流与落盘组** | 《逐级主题汇流块》、ROLLUP、小组空间边界 | F7 |
| **门禁与回声组** | Phase Gate 分型、连续 BLOCK、回声终态、与蜂群 R4 关系 | F8 |

**汇流**：各部门填分表 → **检验统帅**合成 §6 总分与健康度 → 给出 PASS/COND_PASS/FAIL。

## 3. 统一分制与防刷分门槛

### 3.1 细项分值（0–4）

对每个**特性**下的每个**检验细项**（§4 表格中的行）打 0–4 分：

| 分值 | 定义 |
|------|------|
| **4** | 完全达标：该细项全部可核验；**至少一项**关键断言有 **A 级**证据（路径、锚点、表、记录与 CHAIN 对齐） |
| **3** | 基本达标：主路径满足；次要缺口（如部分 GR 子项未显式留痕）不破坏契约 |
| **2** | 部分达标：有明显缺口（缺用户确认、缺 Gate 裁决、穿透不完整等） |
| **1** | 严重缺失：仅口号或大量 B/C 推断冒充已落实 |
| **0** | 未实现或**对抗性违规**（伪造派发、篡改五要素、未确认全开 Task 等） |

### 3.2 特性分 `Score(Fx)`（0–4）

\[
Score(Fx) = \min(\text{该特性下各细项分})
\]

（**最低分制**：防止「一项满分掩盖关键洞」；若某细项 **N/A**，则从该特性 min 计算中**剔除**该细项并注明覆盖数。）

### 3.3 无 A 级时的封顶

- 该细项最高 **2**；若完全无过程痕迹仅模板勾选，最高 **1**。

### 3.4 防刷分 / 判定门槛（R1 转化 · P0 写入）

| 规则 ID | 门槛 |
|---------|------|
| **RG-01** | 每项得分须绑定 **TC 编号**或 **Gate 条目**或 **证据段落 ID**；泛化步骤不得计分 |
| **RG-02** | 仅承认 **本 CHAIN** 且时间在评分窗口内的证据；引用他链须显式「继承」并**减权**或单独计列 |
| **RG-03** | **同一验证动作**不得在 rubric 多维度重复计分；双轨（蜂群 R4 vs 军团 Gate）须在汇总表冻结 **互认 / 重复执行 / 取严** 三选一 |
| **RG-04** | 声称「已提示升级」须 **用户可见载体**（对话摘录/CLI/注释模板）；仅内部口述不得分 |
| **RG-05** | **S/M/L** 须对应可勾选检查项；声称 L3 须附「若降级则删减的义务列表」对照 |
| **RG-06** | **COND_PASS** 是否计入军团分及折损系数须在汇总表冻结，并与蜂群结论一致方可给满档 |
| **RG-07** | **豁免**须逐项映射到维度（哪些强制 0 分、哪些折损）；未映射不得给特性 4 分 |
| **RG-08** | 端到端失败须在汇总表登记 **单一主责维度**；未登记则相关 P0 特性同时压档 |

### 3.5 一票否决（可选触发 · 在汇总表勾选）

若发生以下任一事，可将 **整表裁决为 FAIL** 或相关特性强制 **≤1**：

- 军令状五要素被下级**改写**或 **① 用户原话被摘要替代**；
- 未获用户确认架构即**全量**下发 L3 Task；
- 宣称已创建 Task 但无锚点/代发留痕且标 A 级已派发；
- 蜂群写码回合 **R4 P0 FAIL** 却宣称军团层 PASS。

## 6. 汇总与健康度

### 6.1 简单特性百分化（直观）

\[
S_{raw} = \sum_{x=1}^{11} Score(Fx),\quad
H_{feature} = \mathrm{round}\left(100 \times \frac{S_{raw}}{4 \times N_{eff}}\right)
\]

其中 \(N_{eff}\) = 非 N/A 特性数量（默认 11；若有特性整段 N/A 则剔除）。

### 6.2 加权健康度（推荐 · 防非核心刷分）

| 档 | 特性 | 权重 \(w\) |
|----|------|------------|
| **P0 核心** | F5, F6, F8, F9 | 1.5 |
| **P1 编排** | F1, F3, F7 | 1.2 |
| **P2 治理** | F2, F4, F10, F11 | 1.0 |

\[
S_w = 1.5\sum_{x\in P0} Score(Fx) + 1.2\sum_{x\in P1} Score(Fx) + 1.0\sum_{x\in P2} Score(Fx)
\]

\[
W_{max} = 1.5 \times (4\times 4) + 1.2 \times (3\times 4) + 1.0 \times (4\times 4) = 24 + 14.4 + 16 = 54.4
\]

\[
H_{weighted} = \mathrm{round}\left(100 \times \frac{S_w}{W_{max}}\right)
\]

### 6.3 健康度档位

| 档位 | 条件 |
|------|------|
| **健康** | \(\forall x \in \{F5,F6,F8,F9\}, Score(Fx) \ge 3\) 且 \(H_{weighted} \ge 75\) |
| **亚健康** | 存在 P0 特性 \(Score=2\)，或 \(65 \le H_{weighted} < 75\) |
| **不健康** | 任一 P0 \(Score \le 1\)，或 \(H_{weighted} < 65\)，或 §3.5 一票否决 |

### 6.4 与 PASS / COND_PASS / FAIL 对应

| 裁决 | 条件 |
|------|------|
| **PASS** | 无 §3.5 否决；P0 特性均 ≥2 且均 ≥3 若声称「健康」；TC-LGV-001–008 全过；证据包满足 §1.2 对本轮声称的覆盖 |
| **COND_PASS** | 存在 P1 缺口（如未做双评 TC-LGV-009）或文档化风险；须在汇总表写明保留项与补证期限 |
| **FAIL** | 任一 P0 特性 =0；或 TC-LGV-004/008 失败；或 §3.5 触发；或双轨策略（RG-03）未冻结却混用结论 |

### 6.5 评分争议顺序

1. 以 **本 rubric 版本** 为准；2. 以 **冻结 CHAIN 与证据包** 为准；3. **A > B > C**；4. 冲突时**取严**或请用户裁决（须在表中记录）。
