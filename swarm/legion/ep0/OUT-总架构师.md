## 边界

- **域内（演习可建模）**：多城市群应急指挥、跨机构资源调度、多端态势呈现（指挥席 / 移动 / 大屏等抽象端型）、事件—任务—资源三元关系的时间线视图、预案与波次推演数据流。**可信度：B**（域模型来自行业通识与演习假设，非某单一客户已验收契约）。
- **域外（军令状已冻结）**：不实接政务内网、不假设取得实战资质；本链**默认**不进入应用源码写码与生产部署。**可信度：A**（直接对齐军令状 §④）。
- **隐私边界**：所有示例数据、演示剧本、日志样例须满足「最小必要、分级脱敏、可审计访问」；**禁止**将可识别自然人轨迹、精确到人的敏感细节作为默认可交付示例。**可信度：A**（对齐 [PENETRATING]）。
- **多租户边界**：租户/机构维度的隔离、加密与审计须在**架构层**有单一事实源（策略定义、分发、验证闭环），不得仅在某一客户端或某一集成层「口头等价」。**可信度：A**（对齐 [PENETRATING]）。
- **离线/弱网边界**：允许定义「连接态 / 降级态 / 仅本地缓存态」的**行为契约**与数据新鲜度上限；不要求在本演习链内证明与真实公网 SLA 一致。**可信度：C**（工程假设，待后续环境矩阵标定）。

## ADR 列表

### ADR-001 — 安全与数据策略单一事实源（Policy / Crypto / Audit SSOT）

- **决策**：在逻辑架构中设立**策略与信任根**层：租户/机构策略、加密策略、审计策略由此定义，经网关或侧车组件下发到 API、数据访问层与集成适配器；禁止各端复制互斥规则。
- **状态**：Accepted（演习架构冻结方向）
- **后果**：集成工程师与安全架构师的交付物必须能映射到同一策略模型；增加统一策略变更的发布与回滚流程成本。
- **可信度：A**（直接落实军令状多租户穿透要求）。

### ADR-002 — 态势与调度读写路径分离（CQRS 倾向，非实现绑定）

- **决策**：指挥态势（读模型）与资源调度指令（写模型）在架构上**分离入口与一致性预期**：读侧允许最终一致与显式「数据截至时间」；写侧强一致仅在调度事务边界内保证。
- **状态**：Accepted（纲要级）
- **后果**：离线/弱网场景下读侧可依赖缓存快照；写侧需冲突检测与幂等标识（由后续数据契约冻结）。
- **可信度：B**（行业常见模式，具体一致性级别待 RTO/RPO 与产品拍板）。

### ADR-003 — 离线/弱网降级契约优先于「全功能等价」

- **决策**：定义分级降级：**L0** 全在线；**L1** 只读最近同步 + 禁止远端敏感导出；**L2** 仅本地队列记录意图，恢复后补发；**禁止**在降级态默认可识别自然人全轨迹回放。
- **状态**：Proposed（待产品总监与 UX 纳入验收表）
- **后果**：多端需统一展示「当前模式」与能力边界，避免指挥误判。
- **可信度：B**（架构假设 + 与隐私条款对齐的约束组合）。

### ADR-004 — 示例与夹具数据一律「合成默认」

- **决策**：文档、演示、测试夹具默认采用合成标识符与聚合地理网格；若需轨迹类场景，仅使用**不可回指个人**的统计或匿名化路径。
- **状态**：Accepted
- **后果**：集成与 SRE 的回归环境不得依赖真实个人轨迹种子数据。
- **可信度：A**（对齐 [PENETRATING] 禁止项）。

## 风险

1. **策略 SSOT 与交付节奏脱节**：若产品波次先于策略模型冻结，易出现「端上临时规则」侵蚀单一事实源。**缓解**：将策略模型列为 EP 早期 Gate；合规/安全 EX 产出约束包进 DAG。**可信度：B**。
2. **弱网下调度重复提交**：离线队列补发可能导致重复派单或资源超配。**缓解**：指令幂等键 + 服务端去重为架构必选（细节留给数据平台契约）。**可信度：B**。
3. **多端态势时间戳不一致**：大屏与移动端展示同一事件但时间基准不同，引发指挥误判。**缓解**：全局逻辑时钟/事件版本号在读模型中强制展示。**可信度：C**（假设性风险，依赖实现选型）。
4. **演习范围膨胀**：若后续悄悄要求实网或真实内网数据，将与军令状冲突。**缓解**：任何环境升级须新任务链与用户明示。**可信度：A**（流程风险，对齐军令状）。

## decisions_log

### 已决策

1. 安全、租户隔离与审计策略在架构层维持**单一事实源**，并通过统一下发面覆盖 API 与集成适配器（ADR-001）。**A**
2. 态势读与调度写**路径分离**，并接受读侧最终一致与显式时效展示（ADR-002）。**B**
3. 离线/弱网采用**分级降级契约**，且降级态不开放默认可识别自然人轨迹能力（ADR-003）。**B**
4. 全部示例与夹具默认**合成/脱敏**数据策略（ADR-004）。**A**

### 排除项

1. 本演习链内**不包含**真实政务内网对接与生产级资质假设（军令状非目标）。**A**
2. 本纲要**不**约定具体编程语言、框架、函数级模块划分或仓库目录结构（军令状默认不写码）。**A**
3. **不**将某一厂商专有中间件列为唯一必选；适配层保持可替换假设。**C**
4. **不**在架构纲要中嵌入可识别自然人轨迹的默认业务样例。**A**

### 理由

1. **穿透条款**要求隐私最小化与可审计访问，合成默认与禁止默认识别轨迹可降低合规与声誉风险。**A**
2. **多租户单一事实源**可降低策略漂移与跨端不一致，便于审计与事后追责。**A**
3. **CQRS 倾向与降级优先**契合应急场景高峰读流量与间歇断连的现实约束，避免「伪在线」误导指挥。**B**
4. 排除实网与源码级设计，避免本链越权改写军令状范围，保持 DAG 可执行与回声校验可收敛。**A**
