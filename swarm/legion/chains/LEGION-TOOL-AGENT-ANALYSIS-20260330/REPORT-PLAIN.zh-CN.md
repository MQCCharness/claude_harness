## 你其实问了两个问题

**问题一：** Claude 相关产品里的「子代理（sub-agent）」之间是怎么通信的？为什么写一个很短的 agent 描述文件，就能拉起很多个代理？能不能举个思路上的例子？

**问题二：** OpenClaw 这种要跑很久的智能体，是怎么对付「对话越长越容易忘、上下文装不下」这类问题的？（你原文里的「诗意」这里按**上下文/失忆**来理解。）

下面分开说。

---

## 一、Claude 子代理：它们并不是在「互相发微信」

可以先把关系想成**一个班长 + 多个临时小组长**：

- **父代理（主会话）** 负责拆任务、决定要不要叫子代理。
- **子代理** 各自在**新的、独立的会话**里干活；中间用了什么工具、中间结果怎样，主要留在**子代理自己的会话**里。
- 子代理干完以后，通常把**最终结果**交回给父代理；父代理再决定下一步对用户说什么、要不要再派别的子代理。

所以，**子代理和子代理之间，并不是官方文档里描述的那种「对等直连通道」**。你能观察到的协作方式，更像是：**一切都绕父级编排**——父级去叫人、收作业、再汇总。  
（如果团队用共享文件夹、共享仓库来协作，那是**人的工作方式**，和「SDK 里会话是否隔离」是两层事，不要混成一回事。）

**等级心里有个数：** 上面这套「独立会话、只把最终结果回父级、可以并行多个子代理」的说法，在 Anthropic 公开的 **Claude Code / SDK 子代理文档**里能直接找到对应段落，属于**可以指着链接给别人看**的那一类结论。具体客户端内部怎么实现的细节，没有公开源码就不要当成「铁证」。

---

## 二、为什么「一小段描述」也能启动很多代理？

把「每个代理是啥、擅长啥」写清楚，本质是在做一张**花名册**：

1. **登记：** 每个子代理有一个名字、一段**说明自己适合干什么**的文字（`description` 之类），再加上提示词、允许用的工具等；也可以按约定把说明写在 **Markdown 文件**里（文档里会提到类似 `.claude/agents/` 这种放法）。
2. **选人：** 父会话里的模型根据当前任务，对照这些说明，决定**叫哪一个（或哪几个）子代理**；用户也可以直接点名。
3. **开工：** 每叫一次，就起一个**新的子会话**去跑；几个子任务互不耽误时可以**并行**。
4. **收工：** 子代理把**最终结果**交回来，父级**合并、总结**，再往下走。

用伪代码打比方就是：先 `load` 一堆规格，再 `for` 循环里 `spawn`、最后 `merge`。  
**不是说**磁盘上有个魔法文件会自动变成很多进程——而是**运行环境**读这些短描述，按规则一次次**创建隔离的子会话**。

## 三、OpenClaw：长时间运行靠什么「不忘事」？

OpenClaw 的思路很直白：**真正算「记住」的，是写进磁盘上的 Markdown；对话窗口只是临时工作台。**

文档里大致这几块（都可以在官网读到）：

1. **长期备忘 `MEMORY.md`**  
   放比较稳定的事实、偏好、约定。**每次一对一（DM）会话开始会加载**，像「开机先读备忘录」。

2. **按天的笔记 `memory/年-月-日.md`**  
   记当天的流水、观察。**会自动加载今天和昨天**，让最近发生的事不用全靠超长对话带着走。

3. **两个工具：`memory_search` / `memory_get`**  
   用来搜、读这些文件；配置了嵌入模型时，搜索可以是**语义 + 关键词**混合，避免只会「模糊搜」或只会「精确搜」其中一种。

4. **压缩对话之前，先「冲一把记忆」**  
   在系统要做 **compaction**（把长对话压成摘要）**之前**，会先跑一轮静默提醒：把还飘在对话里的重要东西**尽量先写进记忆文件**，减少「一压缩，关键事实没了」的情况。默认是开的。

**一句人话：** 窗口满了就摘要，摘要前先催你（代理）把该落的笔记落到文件里；长期靠 `MEMORY.md` 和按日笔记撑住；要找旧账用搜索工具。

**等级心里有个数：** 这套机制在 **OpenClaw 官方 Memory / Compaction 文档**里写得很清楚，我们做过**对着官网逐条核对**（不装软件也能核对文字）。**没跑过你机器上的安装和命令**，所以不叫「在你电脑上实测通过」，只叫「文档说得通、链接对得上」。

## 四、这份「分析」可靠到什么程度？

| 说法 | 什么意思 |
|------|----------|
| **能指着官网念的** | OpenClaw 记忆、flush、compaction 几条主线；Claude 子代理「独立会话、结果回父级、可并行、可用描述/文件定义」——都有公开文档链接。 |
| **要你自己多留心的** | 不同产品形态（网页、桌面、API）文档是否完全一致、你用的版本是否改了行为——需要你对照**当前**官方页面或自己试一把。 |
| **我们没做的** | 没有在本任务链里要求「在你电脑上安装 OpenClaw 并跑通命令」；那属于另一类验收。 |

## 五、如果你只想记三句话

1. **Claude 子代理：** 更像「父级派活、子级在独立会话里干完交卷」，不是子代理之间私下组网聊天。  
2. **多启：** 短描述 = 花名册；运行时按名册一次次创建子会话，可并行，最后父级合并。  
3. **OpenClaw 长时：** 记忆在 **Markdown 文件**里；对话要压缩前先 **flush 进文件**；再配合搜索工具把旧笔记捞出来。
