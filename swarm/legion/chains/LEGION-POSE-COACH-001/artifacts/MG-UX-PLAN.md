## 2. 《UX 执行计划》

### 2.1 页面清单（MVP）

| 序号 | 页面/壳 | 目的 | 与军令状映射 |
|------|---------|------|----------------|
| P0 | **启动/首页** | 价值一句话、进入主流程、入口合规摘要 | ② |
| P1 | **轻量信息收集** | 场景/目标、可选偏好 | ② |
| P2 | **相机引导主界面** | 实时预览 + 示例姿势 + 至少一种明确引导 | ②⑤ |
| P3 | **示例姿势详情/切换**（可与 P2 同页不同态） | 更换参照图、简短要点 | ②④ |
| P4 | **权限与隐私说明** | `scope.camera` 场景化说明；数据流与 OP-隐私包一致 | ③、⑤ |
| P5 | **设置/关于**（可极简） | 开关占位、版本、反馈入口 | ③⑤ |
| P6 | **纯示意模式说明**（可 Modal） | 无 VK 或低置信度时的预期与操作 | ④⑤ |

### 2.2 关键用户流程（文字化）

- **A 主路径**：P0 → P1 → 授权 → P2（示例区 + 相机区 + 文案提示）→ P3 可选切换姿势。  
- **B 权限拒绝**：非白屏；场景化说明 + 重试/去设置。  
- **C 人体失败/低置信**：内联或 P6；示意/粗匹配说明；仍可对照拍摄。  
- **D 上传/分析（若启用）**：须独立明示同意；默认路径不假设上传。

### 2.3 信息架构层级

L1 首页价值 → L2 收集 + 拍摄引导（核心）→ L3 示例/帮助/隐私 → L4 错误与降级态。

### 2.4 提示 UI 分层

| 层级 | 区域 | 约束 |
|------|------|------|
| L-参照 | 示例图区 | 不遮挡相机安全区；小屏可折叠 |
| L-实时 | 相机区 | 预览为主；叠加须可读 |
| L-语义 | 文案提示区 | 短句；非长教程 |
| L-反馈 | 粗匹配反馈 | 避免「精确评分」暗示 |

### 2.5 空状态与降级态清单

E1 无权限；E2 曾拒二次进入；E3 相机错误；E4 未检出人体；E5 低置信；E6 示例加载失败；E7 本地 only 不暗示上传云分析。

### 2.6 与军令状⑤对齐的 UX 验收钩子

H1：场景 + 授权后示例+预览+引导同屏。H2：人体失败可理解降级。H3：隐私文案与实现字段级对表（OP-UX 文案表）。
