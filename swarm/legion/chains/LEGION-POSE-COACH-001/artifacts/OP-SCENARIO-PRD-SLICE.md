## 用户故事列表

> 验收口径为**产品可测描述**；具体控件与文案由 OP-UX 落地。负例列于各条「负例/边界」。

### US-001 — 场景化相机权限说明

- **叙述**：作为用户，在需要使用相机前，我希望看到与**当前使用场景**相关的权限说明，以便理解为何需要 `scope.camera`（及平台要求的关联能力）。
- **验收口径（正例）**：
  - 发起相机能力前，存在**与场景挂钩**的说明入口或首屏说明（非通用空话）；用户可继续授权或暂不授权。
  - 说明内容与军令状③「权限与透明」一致，**不**误导儿童/青少年策略（首期可声明为占位或非目标，但须**诚实**）。
- **负例/边界**：
  - **无权限**：用户拒绝或系统未授权时，有明确状态说明 + 可理解的下一步（如前往设置/重试），**不**白屏、**不**静默无响应。
  - 用户曾授权后**撤销**：再次进入时重新触发说明与授权路径，行为一致可预期。

### US-002 — 选择拍照场景（最小必要）

- **叙述**：作为用户，我希望在拍照前选择一个**场景**（或等价筛选），以便只看到与当前拍摄意图相关的示例姿势。
- **验收口径（正例）**：
  - 至少一种「场景 → 姿势列表」的筛选路径；所选场景与 `poses_manifest` 中 `sceneTags` **可对表**（见下文「场景标签冻结表」）。
  - 「信息收集」保持**轻量**：仅收集完成场景筛选所必要的选项，不强制无关个人信息。
- **负例/边界**：
  - **资源失败**：manifest 不可读或为空时，有错误说明 + 恢复路径（重试/返回/联系反馈占位），**不**无限加载。
  - 某场景下**无匹配 pose**：有空态说明（非空白列表无声失败）。

### US-003 — 示例姿势 + 实时预览同框可理解

- **叙述**：作为用户，在已授权相机后，我希望同时看到**示例姿势**与**实时预览**，以便对照模仿。
- **验收口径（正例）**：
  - 选定 pose 后，界面同时呈现：包内授权示例图（或剪影等降级图）与相机预览区域；布局层级由 UX 定，但**对照关系**可感知。
  - 对齐军令状⑤：完成授权后可见示例与预览。
- **负例/边界**：
  - **资源失败**：示例图路径无效或加载失败时，提示 + 降级（如仅文字要点/剪影占位/返回选 pose），**不**崩溃。
  - **无权限**：预览不可用，示例仍可浏览或统一进入「仅示意」说明（由 UX 与工程一致化），但不得假装已开启相机。

### US-004 — 至少一种明确引导（文字或图示叠加）

- **叙述**：作为用户，我希望收到**至少一种**明确的姿势引导（文字提示层级，和/或图示叠加层级），以便调整身体姿态。
- **验收口径（正例）**：
  - 每条可选 pose 在交互流程中可触发或常驻展示**一种以上**引导形态之一：结构化短提示（来自 `referenceHints` 映射的信息架构，非本切片逐字稿）、和/或图示/轮廓叠加。
  - 引导内容与当前选中 pose **一致**，不串题。
- **负例/边界**：
  - **低置信 / 粗匹配不可用**：不承诺高分或实时精准评分；引导降级为**静态示意 + 文字要点**，用户可感知「示意模式」与完整能力差异（对齐军令状④）。
  - **无 VK**：不依赖人体关键点的引导仍可用（纯示意 + 文案层级）。

### US-005 — 人体检出失败时的可理解降级

- **叙述**：作为用户，当设备或运行时不支持人体检出、或检出失败时，我希望进入**可理解的降级模式**，而不是白屏或无任何说明。
- **验收口径（正例）**：
  - 对齐军令状⑤：失败时有用户可读的说明或状态（如「当前以示意模式运行」类信息架构描述）。
  - 仍保留 US-003/US-004 中不依赖检出的最小体验（示例 + 预览或仅示例 + 文字）。
- **负例/边界**：
  - **无 VK**：全程不调用不可用 API 导致崩溃；降级路径可重复进入、稳定。
  - **低置信**：若产品选择展示「匹配反馈」，须避免虚假确定感（例如不宣称「已完美匹配」）；具体措辞边界由 OP-UX 与合规叙事共同收敛。

### US-006 — 隐私与数据流：默认本地、上传可控

- **叙述**：作为用户，我希望清楚知道相机画面**默认如何处理**，以及是否存在上传；若存在上传，须**我同意**且**可关闭**。
- **验收口径（正例）**：
  - 对齐军令状③ `[PENETRATING]`：隐私/数据流说明与真实行为一致；**禁止**默认上传可识别肖像原图用于「改进产品」且未单独明示同意。
  - 若 MVP 存在任何可选上传（如诊断日志、脱敏元数据），须：**单独开关或流程内同意**、最小必要、可关闭。
- **负例/边界**：
  - 用户**未同意**上传：不发生非本地必要传输（与实现一致，验收对照自查清单）。
  - **无权限**：不将「未获得预览帧」误传为「已分析并上传」。

### US-007 — 青少年/儿童占位不误导

- **叙述**：作为监护人或青少年用户，若看到与年龄相关的提示，我希望其**不虚假承诺**完整保护策略（首期可为占位）。
- **验收口径（正例）**：
  - 对齐军令状③：若涉及儿童/青少年文案，仅为占位则须**可读地**标明或为中性通用隐私说明，**不**暗示已具备完整青少年模式。
- **负例/边界**：与 US-001/US-006 交叉：权限与数据说明不与年龄占位**矛盾**。
