# 组织架构方案图（已冻结）— LEGION-POSE-COACH-001

**冻结时间**：2026-03-27  

## 树形结构

```
军团统帅（主对话）
├── 产品与用户旅程部 — 场景定义、 onboarding、留存与指标体系
│   ├── EX-产品战略官 — 纲要 + 对下级任务书 → 末端：蜂群小队
│   └── OP-场景与需求落地员 — PRD 切片、用户路径、验收口径 → 蜂群
├── 体验与引导设计部 — 拍摄页信息架构、提示 UI、无障碍与文案
│   ├── MG-交互与内容设计主管 — 执行计划 + 任务书 → 末端：蜂群小队
│   └── OP-UX/文案执行员 — 线框、组件规范、提示话术 → 蜂群
├── 小程序与视觉工程部 — Camera、VisionKit、性能、骨架匹配 MVP
│   ├── MG-端侧工程主管 — 技术选型闸门、分包与权限 → 末端：蜂群小队
│   └── OP-小程序开发员 — 页面、VKSession、对比逻辑、降级策略 → 蜂群
├── 姿势素材与运营部 — 示例图/剪影、分类、推荐与更新管线
│   └── OP-内容管线员 — 素材规范、本地包、后续 CMS 接口 → 蜂群
└── 隐私与合规参谋（横切）— MG
    └── OP-隐私与权限文案员 — 授权说明、数据最小化、青少年模式预留 → 蜂群
```

## 信息流向

- **产品 OP** → 为 UX OP、工程 OP、内容 OP 提供「场景与验收口径」输入（经统帅 I/O 路由）。
- **UX OP** → 工程 OP：页面结构、提示形态、错误/空状态文案。
- **工程 OP** → UX OP：VisionKit 能力边界、帧率与耗电风险（反哺文案与动效强度）。
- **内容 OP** → 工程 OP：示例图路径、元数据（标签、关键点参考姿态 JSON 若采用）。
- **隐私 OP** → 全部末端：授权弹窗、设置页说明、数据分析默认关。
