## 纲要

1. **默认行为不变**：无新 flag 时，导出路径与单文件 HTML 的**默认外观**与当前版本一致（浅色/无主题切换视为现状）。  
2. **暗色主题为可选**：通过**显式** CLI 开关或等价配置启用；推荐同时支持 `prefers-color-scheme` **仅当**用户显式选择「跟随系统」模式（若实现分两阶段：先纯手动暗色，后跟随系统，须在 MG 层标为非目标/阶段）。  
3. **页脚版权行**：仅影响 HTML 内**指定插槽**；不得改写 Markdown 源文件；字符串必须经过 MG/OP-FOOTER 定义的安全策略。  
4. **无外部服务**：主题与页脚均不得触发网络请求（含字体 CDN 若当前无则禁止新增）。

---

## 兼容策略

- **CLI**：新增 flag 须为**附加式**；旧脚本零参数调用路径不变。  
- **HTML**：主题通过 `class`/`data-*` 作用于根节点或 `<body>`，不破坏现有正文 DOM 结构；页脚为独立块级区域。  
- **降级**：未知终端/无 TTY 不影响导出；主题仅影响内联/CSS，不依赖 JS 运行时（除非现有工具已依赖；若已依赖则保持现状不扩大）。
