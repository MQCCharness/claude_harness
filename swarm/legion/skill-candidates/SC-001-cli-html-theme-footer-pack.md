## 可复用套路（候选技能核心）

- **触发条件**：内部 CLI 将 Markdown 转为**自包含单文件 HTML**，需增加**展示层**（主题）与**用户可控字符串**（页脚）且约束「无新外部服务、路径契约不变」。  
- **操作步骤**：  
  1. EX 冻结：默认 CLI 不变；新 flag 仅附加；禁止外链依赖。  
  2. MG 冻结：HTML 壳层插槽（`#export-footer`）与主题根钩子二选一（`html` 类或 `body data-theme`）。  
  3. OP-THEME：CSS 语义变量 + 暗色覆盖块；代码高亮与页面主题成对设计；R2 对齐 WCAG 对比度分层（1.4.3 / 1.4.1 / 1.4.11）。  
  4. OP-FOOTER：默认 **实体转义或 `textContent`**；`--footer` 与 `--copyright-file` **互斥 fast-fail**；分 `NOT_FOUND` / `READ` / `ENCODING` / `SOURCE_CONFLICT`。  
  5. R4：默认无参基线快照 + 暗色 flag + XSS 字符串三者为 P0；无实现时 **COND_PASS** 并声明重跑条件。  
- **产出物**：`MISSION-BRIEF` 级契约 + `TC-B-*`/`TC-E-*` 表 + `decisions_log` + R4 裁决一行。  
- **禁忌 / 反模式**：页脚默认 `innerHTML` 直灌；主题与页脚职责交叉改 DOM；未定义 CLI 冲突优先级。
