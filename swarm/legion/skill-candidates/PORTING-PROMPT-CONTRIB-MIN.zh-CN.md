# 移植主提示词：众人拾柴（贡献沉淀 → 用户升格）— 最小可落地版

> **用途**：复制到**没有**本仓 `INDEX.md` / `SC-NNN` / `WAVE-CLOSE` 全套机制的宿主；只保留**能保证特性生效**的底线，其余可删或映射到宿主自有工具（工单、Wiki、Notion、单文件日志等）。  
> **本仓完整版指针**：`03_AI行为准则/21_军团众人拾柴与技能候选沉淀.md`（Cursor + `hajimi-dog` 专用）。

---

## 0. 执行者与目标

**编排者（统帅）**：多角色 / Sub-agent / 分阶段任务的总线。  
**目标**：作战中把**可验收、可复用、有利于冻结目标**的套路，从聊天里拽到**持久载体**；在**阶段结束**或**总目标收敛**时，**必须询问用户**是否并入宿主**正式知识库**（技能、Playbook、Runbook、规范树——名称由宿主定）。

---

## 1. 三条硬不变量（删啥都不能删这三条）

1. **持久化**：贡献不能停在口头；须落在宿主选定的**单一真源**（见 §3），可引用、可 diff。  
2. **用户升格闸**：凡存在「建议并入正式库」状态（宿主可改名，如 `READY` / `待采纳`），**阶段收口前**须完成 **§5 提问** 并**留痕**（摘录、链接、消息 ID 均可）；否则**不得**宣称本阶段治理闭环完成。  
3. **正式库与草稿分离**：**禁止**未获用户表态就把草稿写入**对外权威的**规范/技能根目录；合并须走宿主既定的评审/PR/所有者流程。

---

## 2. 贡献一条「众人拾柴」至少要有什么（最小字段）

| 字段 | 说明 |
|------|------|
| **何时/哪一阶段** | 例：第几轮、哪类子任务、哪条链 ID（无则写 `NONE`） |
| **子任务摘要** | 一句人话：派工做了什么 |
| **对目标的帮助** | 对齐哪条成功标准或里程碑 |
| **可复用动作** | ≥3 步或可触发条件 + 预期产出 |
| **证据** | 路径、条目号、命令+退出码摘要其一；没有则全文标 **B/C** |
| **状态** | `DRAFT` → `READY_FOR_REVIEW`（或宿主等价）→ `MERGED` / `DECLINED` |

**反例不进「柴」**：纯搬运、无泛化价值；**失败教训**进「坑库」通道，勿与可复用套路混桶（若宿主无坑库，用单独标签 `LESSON_FAIL`）。

## 4. 何时写（触发）

- 子任务 **通过验收** 且能拆出可复用步骤 → **可**记（不必每条都记）。  
- **阶段门禁通过**后，合并本阶段亮点 → **可**记一条。  
- **阶段收口**：在收口物中增加 **「本阶段贡献：`C-xxx` 列表或 `N=0`」**（无新增则必须 **显式写零**）。  
- **目标收敛**：对状态为 **READY_FOR_REVIEW** 的条目跑 **§5**。

---

## 5. 用户裁决话术（可缩短，语义不变）

> **【众人拾柴】** 当前贡献库有 **N** 条记录，其中 **M** 条待您决定是否纳入正式知识库。请对每条 **READY** 从以下信息判断：**阶段、子任务、效果、证据** —— **采纳并入 / 合并进某现有文档 / 仅留草稿 / 弃用**。  
> 若您说 **「本次全部不升格」**，也请**显式留痕**，以便下一阶不再重复追问同一批。

**执笔**：仅 **统帅**（或宿主指定的单一 Owner）；执行子任务的角色**不得**直接改正式库。

## 6. 「真落地」P0 自检（宿主剪裁后至少保留）

| ID | 期望 |
|----|------|
| P0-1 | 任意新成员能找到贡献真源的**入口**（README 一行或导航页） |
| P0-2 | **阶段收口**出现 **贡献列表或 N=0**，且与真源可对账 |
| P0-3 | 存在 **READY** 时，收口线程/文档中有 **§5 提问或用户答复** 痕迹 |
| P0-4 | 无用户表态则**无**对正式权威文档的静默写入 |
| P0-5 | 单条贡献含 §2 最小字段中的**可复用动作 + 证据或 B/C** |

## 7. 降级（仅聊天/无仓时）

若**不能**落盘：在**用户可见**的长期线程用 **一条置顶消息**维护表格（日期 | 摘要 | 状态 | 消息链接），并声明 **「本链为 L0-聊天真源，迁仓后须导入 CONTRIBUTIONS」**。**不得**同时声称已满足 **§1 三条硬不变量** 中的「持久化」至仓库层面。
