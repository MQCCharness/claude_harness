# 众人拾柴 · 技能候选区（Legion × Skills）

本目录存放**军团/蜂群作战过程中**产生的 **Skill 候选**（非正式技能）：可验收的子任务若沉淀出**可复用套路**，由统帅按模板落盘；**是否升格**入 `.claude/skills/hajimi-dog/` 由用户在**波次间隔**或**收敛后**拍板。

**权威操作说明**：`.claude/skills/hajimi-dog/03_AI行为准则/21_军团众人拾柴与技能候选沉淀.md`  

**异构宿主移植（精简，无全套索引）**：`PORTING-PROMPT-CONTRIB-MIN.zh-CN.md`（同目录）。

**闸口（与渐进闭环对齐）**：`INDEX.md` 中若存在 **`READY_FOR_REVIEW`** 行，**波次收口**必须触发 **§6 用户裁决**并留痕，否则 **`PORTING` §3 U6** 为真、渐进闭环 **P0** 不通过。本波 **无新增 SC** 时，收口包 **F** 须写 **`N=0` 显式零申报**。

## 文件约定

| 文件 | 用途 |
|------|------|
| `INDEX.md` | 候选总表（`SC-NNN`、CHAIN、阶段、评分、路径） |
| `SC-NNN-短slug.md` | 单条候选正文（模板见 `CANDIDATE-TEMPLATE.md`） |
| `CANDIDATE-TEMPLATE.md` | 复制用模板 |

## 与正式 Skills 的区别

- **候选**：实验/复盘级，**不**参与 `skills_linter.py` 全量门禁，直到用户同意升格。  
- **正式技能**：须走 `02_技能体系维护与进化.md`，更新 `00_全局索引`，并 **运行 linter**。

## 禁止

- 写入密钥、token、完整隐私请求体。  
- 把一次性的项目指纹包装成「通用技能」而不标 **B/C**。  
- 未经用户明示将候选**擅自**合并进 `hajimi-dog` 树。
