# 小组信息空间（Group Space）

**语义**：在**同一领导角色辖内**（冻结架构子树），执行期共享的**组内共识区**；**不穿透**到兄弟部门，**不替代**军令状与逐级汇流契约。

## 路径模式

```
group-space/<CHAIN>/W<波次>/<TIER>-<领导角色slug>/CONSENSUS.md
```

- 可增加 `ARTIFACTS.md`、`GLOSSARY.md` 等，保持单链可审计。
- **新 CHAIN** 或 **推翻架构** 须 **新目录**，禁止静默复用旧 `CHAIN` 路径冒充同一契约。

## 写入边界（优先级）

1. 军令状五要素、**`[PENETRATING]`** — **高于** 组空间任意条文。  
2. 组空间 — **高于** 组内口头拍板但未落盘的**个人笔记**（若与组空间冲突，以组空间为准并标 **B** 待上级裁剪）。  
3. 组空间内容**不得**默认升格为 **跨组 A 级事实**；须经 **军团统帅** I/O 裁剪或 Echo 对账。

## 与派发、汇流的关系

- **《下级 Task 发包》**：仍须独立存在；组空间**不可替代**发包最小信息集。  
- **《逐级主题汇流块》**：必填字段不可省略；可在「上送指针」中增加 **组空间路径** 作为补充阅读。  
- **叶 OP**：若直连下级仅为 **L2 蜂群**，组空间可由 **OP 作为 +1** 维护「蜂群组内共识」子页（同路径模式，`TIER-OP`）。

## 波次与收口

在 **`WAVE-CLOSE-*`** 中声明：本波使用到的组空间目录列表，或 **`组空间 N/A`**。COND/开放假设须在 Gap 或残差表留痕。

## 权威规则锚点

`CLAUDE.md §三`「小组信息空间」专节。
