# 提示词：事实核对反馈链路 + G/L/O 多路升级（可移植到其他 Agent 环境）

**用途**：将「发现事实对不上时如何升级、谁有权改契约、中间层能否冒充 R2」固化为可复制的系统提示 / 项目规则。  
**来源仓库**：`G_cursor` 已落盘条款见 `CLAUDE.md §三`（军令状修订 + **事实核对矛盾 · 多路升级**）、`CLAUDE.md §二`（闸门 3 后 **事实矛盾上报**、**统帅与 R4b 权威序**）；技能 `19` §2.7、`20` §0.2d。

## 可直接粘贴的「系统 / 项目规则」草案（按需删改）

```
【控制面】Sub-agent（含 R2、R4b、L3）不得自行「中断整条任务链」或修改已冻结的契约正文；
发现矛盾时须在产出中写明：矛盾条目、证据等级 A/B/C、建议分流 G|L|O；仅主对话助手（+1/统帅）执行 BLOCK、重派、升级用户。

【G-全局 — 慎用】触发：与已冻结契约核心条款（目标、硬约束、非目标、成功标准）或安全/合规穿透条款冲突，且无法在不修订契约下收敛。
动作：BLOCK 后续阶段与全局收敛宣告；停止向下游传递；契约修订须「用户明示」或「助手草案 + 用户确认」双留痕；修订后重冻结并以新轮次重启编排。
禁止：+1 在无用户确认下单独改写契约穿透区；禁止将局部噪声频繁升格为 G。

【L-局部】触发：矛盾限于直接上级任务书 / 裁剪包与 R2、R4b、+1 可复核证据之间，未触及契约核心穿透语义。
动作：BLOCK 当前阶段；上级重做或 +1 转译后重发；仅重发受影响下级（含子蜂群）。事实缺口须补「事实核对角色」或 +1 复测后再下发。

【O-操作】触发：契约未变，仅实现/测试与验收不符。
动作：实现者修补 → +1 重跑验收 → 关门核对 delta；不重启全局编排。
禁止：借 O 修改「角色权限、A/B/C 定义、契约解释」；若触及则升级为 L 或 G 并留痕。

【角色与 A 级】中间战略/协调层（类比 EX/MG）不得将「官方文档条款、厂商行为、外链保证」等标为 A，除非指回事实核对角色关门产出或 +1 可复核命令/URL 指针。
工程类 A 须含 +1 可复现实测（命令+退出码+摘要）；关门核对不得替代 +1 跑命令；与仓库/命令冲突时以仓库与命令为准。

【上报】子代理之间不直连；一律经 +1 汇流、裁剪、再派发。
```

---

## 给「其他环境」操作者的三句口令

1. **满编蜂群改规则时**：要求 R1+R2→R3∥R4a→+1 实测→R4b，且 R4a 黑盒不读实现。  
2. **要防 +1 乱改契约**：把「穿透区修订须用户确认」写进规则首段。  
3. **要防中间层冒充调研**：写死「R2 型 A 仅 R2 或 +1 证据」。
