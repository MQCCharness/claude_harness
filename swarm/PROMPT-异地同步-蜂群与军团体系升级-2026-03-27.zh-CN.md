**权威**：以**已对齐新语义**的蜂群/军团主规则与技能 **12 / 20** 的**文件字节**为准；若通过 zip/Git 迁移，**优先整包覆盖**这些文件，再用本提示词做 **diff 核对与演示链清扫**。  
**路径**：**不假定**规则、技能与落盘目录与任一固定 layout（例如某 IDE 默认的 `.claude/…`）一致；**一律先按 §3 文件名在项目内搜索定位**，再编辑与验收。

**互指**：同仓库内 **裁决简报**（文件名 `RECENT-DECISIONS.md`，搜索定位）中 `2026-03-27` 相关条目；**体系升级记录**（文件名含 `蜂群体系升级记录`）中 2026-03-27 蜂群 R4a/R4b 节。

## 1. 人读摘要（升级包内含哪些变化）

### 1.1 蜂群（L2 · 文件名 `CLAUDE.md §二`）

- **编队时序闸门**：**R1+R2 关门** 后方能派发 **R3**；**R4a** 可与 **R3 并行**，但 Task 输入 **仅限契约级**，**不得**纳入 R3 成果；**统帅**按 **R4a 冻结表**实测；**R4b** 仅在 **R3 关门 + 证据回灌** 后派发，做关门核对（可含白盒例外流程）。
- **写码回合**：最低触达 **R3 + R4a + R4b**（不再是笼统「R3+R4」一条）。
- **满编（技术最小集）**：**R1、R2、R3、R4a、R4b** 各至少一次 Task（平台可用时）。
- **验收裁决**：**PASS / COND_PASS / FAIL** 定义保留；**COND_PASS** = 全部 **P0 实测通过** 前提下，仍存在仅人工说明的 **P1** 或已文档化剩余风险。
- **Task 锚点**：`ROLE` 须支持 **R4a | R4b**（与满编五段一致）。
- **技能 `12`**：误读表、P0 文档审计等与上述一致。

### 1.2 军团（L3 · 文件名 `CLAUDE.md §三` + 技能 `20`）

- **不得收窄**蜂群底线；写码措辞统一为 **R3+R4a+R4b** 与蜂群时序闸门。
- **Phase Gate 裁决**：**PASS / BLOCK / COND_PASS**（**CONDITIONAL_PASS** 与 **COND_PASS** 同义，新文用 **COND_PASS**）。宣告 **COND_PASS** 须留痕：**COND_ID**、条件说明、负责消化方、消化时限、关联 **TC-ID**（若已冻结）；语义**不得弱于**蜂群 `CLAUDE.md` 对 **COND_PASS** 的定义。
- **叶 OP Gate**：蜂群验收须为 **PASS** 或 **合法 COND_PASS**（可核对蜂群定义与 **COND_ID**）；**FAIL 不得放行**。
- **双层门禁**：蜂群 **FAIL** → 军团不得宣告全局 **PASS**；蜂群 **COND_PASS** → 军团不得仅因「非 PASS」误杀，须核对条件与留痕。
- **回声校验**：结论用语 **PASS / BLOCK / COND_PASS**；与 **TC-B-\*** / **TC-E-\*** 双向锚定；主观满意须 **ACCEPT 记录 + TC-ID** 方可标 **A 级**；与叶 OP 内 **R4a 冻结表**互指。
- **收敛与熔断**：未满足收敛不得宣布完成。**叶 OP**：同一交付目标连续 **5** 次 **R3→R4b** 闭环仍不收敛 → **强制重入 R1+R2**（蜂群闸门），再进入 **R3+R4a+R4b**。**军团**：同一 **CHAIN** 下回声连续 **5** 次仍 **BLOCK** 或未合法 **COND_PASS** → **强制 R1 型 + R2 型** Task（Sub-agent 隔离）或用户同意改军令状/DAG/I/O 后重跑。
- **ECHO-CHECK.md / 回声模板**：推荐列含 **TC-ID、COND_ID、ACCEPT**；技能 `20` 增 **§0.2a–0.2c** 与模板表、§3.3 P0、§4 衔接。

### 1.3 可选后续（非必须）

- 历史演示链、PORTING 文案里仍可能写旧词 **R3+R4** / **CONDITIONAL_PASS**，可按需批量替换；**规范权威**以更新后的**主规则 + 技能 12/20** 为准。

## 2. 可复制长提示词（整段粘贴给目标环境 AI）

```markdown
【任务】你是统帅（+1）。当前仓库声称与「HajimiDog 蜂群 4+1 + 军团 L3」治理栈同源，但可能仍为旧版语义。请完成**体系升级对齐**，使磁盘上的规则与技能与下文「冻结语义」一致。

【路径解析 — 必须先做】勿假定文件在固定目录。请在仓库根下按**文件名**搜索并记录路径，下文「蜂群规则」「军团规则」「技能12」「技能20」「linter」「裁决简报」均指你解析到的文件：
- CLAUDE.md §二
- CLAUDE.md §三
- 12_蜂群规则与技能体系维护指南.md
- 20_军团模式回声校验与规模裁量.md
- skills_linter.py（若存在）
- RECENT-DECISIONS.md

改前用 Read/搜索确认内容与差异，再最小必要地编辑落盘。

【冻结语义 A — 蜂群 L2】
1. 蜂群规则文件须含独立章节「蜂群编队启动与时序闸门」，且包含不可违背的闸门：
   - 闸门1：R1+R2「关门」之前，不得派发 R3 Task（诚实边界分节等效须声明）。
   - 闸门2：R3 启动须满足最小输入（R1/R2 结论已进入 R3 约束或 Task 正文已摘要）。
   - 闸门3：R4a 可与 R3 并行，但 R4a Task **仅限契约级**输入，**禁止**纳入 R3 实现细节；统帅按 R4a 表做实测；R4b **仅在 R3 关门且证据回灌后**派发，做关门核对。
2. 「写码回合最低触达」须明确为 **R3 + R4a + R4b**，不得以「R3+R4」一笔带过而省略 R4a/R4b 时序。
3. 用户要求「满编」时，技术最小集须为 **R1、R2、R3、R4a、R4b** 各至少一次 Task（平台可用时）；Task 首行锚点 `ROLE` 须允许 **R4a|R4b**。
4. R4 清单门禁为三段式：**R4a 冻结清单 → 统帅清单执行（实测）→ R4b 关门核对**；验收裁决 **PASS / COND_PASS / FAIL** 与蜂群规则文件定义一致（COND_PASS 不得放宽为「未跑 P0」）。
5. 技能12中误读表、写码回合与 R4a/R4b 相关条款须与蜂群主规则一致，禁止残留「R4 先读实现再写清单」的旧叙述。

【冻结语义 B — 军团 L3】
1. 军团规则文件须明确：军团**不修改、不覆盖、不收窄**蜂群底线；写码回合用语为 **R3+R4a+R4b** 与蜂群时序闸门。
2. Phase Gate 裁决：**PASS / BLOCK / COND_PASS**；**CONDITIONAL_PASS** 与 **COND_PASS** 同义；**COND_PASS** 须登记 **COND_ID、条件、负责方、时限、关联 TC-ID**，且语义不弱于蜂群 COND_PASS。
3. 叶 OP Gate：仅当蜂群为 **PASS** 或**合法 COND_PASS**（可核对 COND_ID 等）时放行；**FAIL 不得放行**。
4. 双层门禁：蜂群 **FAIL** 不得被军团层面 **PASS** 覆盖；蜂群 **COND_PASS** 不得被军团误杀为无效。
5. 回声校验：使用 **PASS/BLOCK/COND_PASS**；与 **TC-B-/TC-E-** 互指；主观验收须 **ACCEPT+TC-ID** 才能标 A 级；与叶 OP **R4a 冻结表**互指。
6. 收敛：未满足「P0 TC A 级闭环 / PASS / 合法 COND_PASS / ACCEPT+TC」之一不得宣布任务完成。熔断：叶 OP 同一目标 **5** 次 R3→R4b 不收敛须强制 **R1+R2** 重入；军团同一 **CHAIN** **5** 次回声仍 BLOCK 须强制 **R1 型+R2 型** Task 或改契约。
7. 技能20须含 **§0.2a COND_PASS 对齐**、**§0.2b 回声↔TC**、**§0.2c 收敛熔断**；模板表含 **TC-ID/COND_ID**；§3.3 P0 为 **R3+R4a+R4b**；§4 与蜂群 R4a/R4b 衔接。

【执行步骤】
1. 在蜂群规则、军团规则、技能12、技能20中搜索「R4a」「R4b」「蜂群编队启动与时序闸门」「收敛、轮次熔断」等；列出缺口文件。
2. 若有 Git：可从已升级参考副本 **整文件覆盖**；否则按「冻结语义」逐段**补写**缺失章节（保持原有 frontmatter、规范栈指针风格）。
3. 若存在 linter：`python <解析到的 skills_linter.py 路径>`，退出码应为 0。
4. 在裁决简报 **文末追加**一节，标题含日期与「蜂群编队 R4a/R4b + 军团 COND_PASS/收敛熔断 同步」，写清**实际编辑路径**与**可验证依据**（命令/Read 指针）。
5. 向用户输出：改动文件列表、残留 P1（如演示链仍写 R3+R4）、下一轮建议。

【禁止】

- 在 R4a Task 中默认附带完整 diff/整文件源码（除非用户明示白盒并标注例外）。
```
