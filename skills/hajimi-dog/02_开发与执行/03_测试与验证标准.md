# HajimiDog 测试与验证指南 (Check Skills)

## 1. 嵌入式固件 Check Skills

*   **舵机与 PWM 模块检测**:
    *   **方法**: 编写一个简单的平滑扫角测试函数，让所有舵机从 0° 缓慢转到 180° 再转回。
    *   **观察点**: 观察是否有卡顿、异响或剧烈抖动，以此验证 PWM 占空比计算是否准确以及电源供电是否稳定。
*   **蓝牙通信链路检测**:
    *   **方法**: 使用手机蓝牙串口助手连接 HC-05/08 模块，发送测试字符（如 'A', 'B'）。
    *   **观察点**: 在 `main.c` 的串口中断中点亮板载 LED 或在 OLED 打印接收到的字符，确认波特率匹配且中断触发正常。
*   **非阻塞状态机检测**:
    *   **方法**: 在宠物执行一个长周期动作（如"走路"）时，高频发送蓝牙指令。
    *   **观察点**: 如果宠物能立即打断当前动作并响应新指令，说明非阻塞逻辑达标；如果必须等动作做完才响应，说明代码中存在隐式死循环阻塞。
*   **传感器数据轮询与中断测试 (Sensor Test)**:
    *   **方法**: 接入新传感器（如超声波）后，在 `main.c` 主循环或定时器中断中读取数据，并通过串口或 OLED 实时打印。
    *   **观察点**: 观察数据更新频率是否稳定，是否因为读取传感器导致主循环（如舵机 PWM 更新）卡顿。如果出现卡顿，说明读取逻辑存在阻塞（如死等回波），必须改为中断或状态机轮询。

## 2. 可视化上位机 Check Skills

*   **OLED 代码生成精准度检测**:
    *   **方法**: 在可视化界面极端拖拽元素（如拖到屏幕边缘或重叠），观察右侧 `CodePreview` 面板。
    *   **观察点**: 检查生成的 `OLED_ShowString` 或 `OLED_ShowImage` 函数的 X, Y 坐标是否与界面完全一致，且是否正确处理了越界（超出 128x64）的边界情况。
*   **逻辑蓝图连线闭环检测**:
    *   **方法**: 创建一个复杂的"一对多"或"多对一"逻辑链（例如：蓝牙收到 'A' 同时触发切换模式和更新屏幕）。
    *   **观察点**: 检查生成的 `main.c` 中 `while(1)` 循环内的 `if` 嵌套结构是否正确闭合，C 语言语法是否合法，变量名是否拼写正确。
*   **代码生成校验拦截测试 (Validation Test)**:
    *   **方法**: 故意在逻辑蓝图中制造错误，如孤立节点、参数越界（如设置负数距离）、循环依赖。点击"生成代码"按钮。
    *   **观察点**: 检查上位机是否能成功拦截错误，弹出 UI 警告提示，且**不生成无效的 C 代码**。
*   **状态管理 (Zustand) 性能检测**:
    *   **方法**: 快速拖动 OLED 元素，打开浏览器的 React DevTools Profiler。
    *   **观察点**: 检查是否引发了整个 App 的不必要重渲染。确保只有 Canvas 和 CodePreview 区域在更新，侧边栏等无关组件不应重新渲染。
