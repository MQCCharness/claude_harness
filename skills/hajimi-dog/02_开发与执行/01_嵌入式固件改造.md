# HajimiDog 嵌入式固件改造指南

## 1. 新增宠物动作 (修改 `petaction.c`)
*   **流程**: 确定动作周期 -> 拆分关键帧 -> 使用非阻塞延时 (`delay_ms` 或定时器标志) 逐步更新舵机角度。
*   **致命禁忌**: 绝对不能在动作函数中使用死循环 `while` 阻塞主线程，否则会导致蓝牙中断无法及时响应或主循环卡死。

## 2. 修改 OLED 显示 (修改 `oled.c` / `oled_data.c`)
*   **流程**: 使用 PCtoLCD2002 提取图像字模 -> 在 `oled_data.c` 中定义 `const unsigned char` 数组 -> 在主逻辑中调用 `OLED_ShowImage`。

## 3. 增加新传感器/外设 (Sensor & Peripheral Integration)
*   **流程**: 在 `HardWare` 目录下新建 `.c/.h` 文件 -> 配置 GPIO / ADC / I2C -> 在 `main.c` 的 `Init` 阶段调用初始化函数 -> 在主循环中轮询读取数据并更新状态机。
*   **致命禁忌 (非阻塞读取)**: 接入如超声波 (Ultrasonic) 等需要等待时序的传感器时，**严禁使用死循环 (如 `while(ECHO==0)`) 等待回波**。必须使用**定时器输入捕获中断 (Input Capture Interrupt)** 或**状态机轮询**来读取数据，否则会阻塞主循环，导致蓝牙断连或舵机卡顿。
*   **状态机集成**: 新传感器的状态（如 `distance < 10cm`）应作为全局标志位，在 `main.c` 的主状态机 `Action_Mode` 中作为状态切换的触发条件（Trigger），而不是直接在传感器读取函数中执行长耗时动作。
