# HajimiDog 项目架构与工作体系

HajimiDog 是一个软硬结合的桌面宠物项目，整体分为两个核心部分：**嵌入式固件端** 和 **可视化开发上位机端**。

## 1. 嵌入式固件端 (`deskto-pet-code`)
基于 STM32F103 微控制器的裸机 (Bare Metal) C 语言工程，负责宠物的底层硬件驱动与行为控制。
*   **硬件驱动层**: `oled.c` (I2C显示驱动), `pwm.c`/`servo.c` (定时器PWM舵机驱动), `bluetooth.c` (UART蓝牙通信)。
*   **行为逻辑层**: `petaction.c` (步态与动作映射)，将抽象动作转化为具体的舵机角度序列。
*   **主控状态机**: `main.c` 中的 `Action_Mode` 状态机，负责非阻塞调度各个动作和响应蓝牙指令。p

## 2. 可视化开发上位机 (`visual-dev-studio`)
基于 React + TypeScript + Vite 构建的 Web 端可视化 IDE，用于降低嵌入式代码的编写门槛。
*   **OLED 可视化编辑器 (`OLEDEditor`)**: 拖拽式 UI 设计，支持文本和图片，自动计算坐标和尺寸。
*   **逻辑蓝图编辑器 (`LogicEditor`)**: 基于 `@xyflow/react` 的节点式编程，通过连线定义触发器 (如蓝牙指令) 与动作 (如切换状态、更新屏幕) 的关系。
*   **代码生成器 (`store/index.ts`)**: 使用 Zustand 管理全局状态，将 UI 和逻辑蓝图实时编译为符合 STM32 固件规范的 C 语言代码 (`generateOLEDCode`, `generateMainCode`)。

## 3. 前后端数据流与交互模式 (Data Flow & Interaction)
*   **离线编译流 (Web -> STM32)**: 上位机**不直接**通过网络控制硬件。用户在上位机完成设计后，点击"生成代码"，将生成的 C 代码复制并覆盖到 `desktop-pet-code` 工程中，通过 Keil/STM32CubeIDE 编译并**手动烧录 (Flash)** 到单片机。
*   **实时控制流 (Phone/PC -> STM32)**: 固件烧录运行后，用户通过手机蓝牙 APP 或 PC 蓝牙串口助手，向 HC-05/08 模块发送单字符或字符串指令（如 `'A'`, `'B'`），触发 `main.c` 中的状态机切换。
