# 工具成功与 UI「不可见」— 调试清单（蜂群补强 · 非强制条款）

**权威边界**：本文件是 **操作型调试清单与归因框架**，**不**替代 CLAUDE.md 的 Agent 子代理/R4 底线；**不**收窄写码回合与实测门禁。
**通用化**：不写死某一应用仓库、专有工具名或唯一 API 路径；反模式用 **短标签 + `<占位>`**，见 **`12`** 技能通用化短节。

## 1. 为何要单独拆这一条

「**工具成功**」与「**用户可见变化**」在工程上是 **两条可独立失败的契约**。混为一谈时，容易出现：

- 把问题归咎于「模型/上游不行」，而实际是 **视口契约** 或 **渲染过滤**；
- **隔离单测 / mock 层全绿** 仍复现不了「界面不变」，因为测试往往不经过 **真实渲染与布局管线**（分层测试共识，可标 **B**）。

**R1 转化（冻结验收时）**：在「协调摘要 / 工具执行结果」层之外，显式增加「**可见性契约**」或 **TC-B-** 步骤（见 §4）。

## 2. 架构师归咎分层（由高到低排查顺序）

按 **信号路径** 自上而下分流，避免第一次猜就钉在「模型坏了」：

| 顺序 | 层级 | 典型原因（抽象） |
|------|------|------------------|
| A | **产品/视口契约** | 只渲染某一屏、某一图层、某一虚拟列表视窗；多文档/多会话非同一状态快照 |
| B | **客户端渲染规则** | 几何为零或无效跳过绘制、坐标在视窗外、透明/与底色不可分、全量重挂载导致选中/滚动丢失「像没变」 |
| C | **客户端状态同步** | 仅终态刷新、流式/增量协议未在中间态对齐权威状态、鉴权缺失导致拉取失败被静默吞掉 |
| D | **服务端领域规则** | 实体挂在当前视图不包含的容器下、类型与父子策略冲突（仍可能对调用方返回成功语义，依产品定义） |
| E | **工具参数 / 校验层** | 嵌套结构被二次编码为字符串、分片传输导致 JSON 残缺、校验错误信息未映射到「为何不可见」 |
| F | **模型与编排** | 目标容器 id 选错、写入分支与「默认可见区域」不一致 |

### 2.1 常见「匿名反模式 → 层级」速查（示例占位）

以下用 **模式标签**，实施时替换为你的域名实体（须符合 **`12` 通用化纪律**）：

| 模式标签（可检索） | 多落在层级 |
|--------------------|------------|
| **嵌套对象被字符串化**（例：`<payload.nested>` 期望 object 实为 string） | E |
| **仅首屏/默认分支可绘**（例：实体在 `<非默认容器>`） | A |
| **零或负几何**（例：宽高校验导致跳过绘制） | B |
| **仅终态一次提交 UI**（例：未开启增量同步路径） | C |
| **工具参数字符串在流式下损坏**（例：SSE 分片拼接） | E（常与 F 表象重叠） |

## 3. 统帅 / Debugger 最短动作单（可复制）

1. **冻结现象句**：「成功」指什么？（HTTP 状态 / `tool_result` / 业务 `ok` / 版本号 / 副作用字段）
2. **拉一条权威状态**：与当前 UI 会话绑定的 **状态快照**（REST/GraphQL/本地存储导出/响应体中的文档载体—**以你的产品契约为准**）是否与界面声明的版本或哈希一致？
3. **在快照中定位变更**：新实体是否存在？父链是否落在**当前视口契约**允许的容器下？
4. **查渲染与可见性条件**：是否存在 **几何/裁剪/虚拟化** 导致不绘制？
5. **再下结论**：快照中无变更 → 优先 **执行层与参数**；快照有、屏上无 → 优先 **视口与渲染**。

## 4. 与蜂群 R1/R2/R3/R4 的衔接（防再漏）

| 角色 | 建议 |
|------|------|
| **R1** | 撕「**成功仍不可见**」：错容器、零尺寸、离屏、双文档、竞态；每条 **→R4** 一条可测断言 |
| **R2** | 厂商协议、流式分片、**函数调用参数字符串**形态等 **事实**；**→R3** 兼容策略 |
| **R3** | 参数兼容、视口提示、中途同步、日志可观测；**回滚点**写清 |
| **R4** | **TC-B-**（黑盒可见性）与 **TC-E-**（单测）分工；P0 至少一条 **端到端或录屏夹具**，避免「仅 mock 工具层」冒充全链路 |

## 5. 自动化补强方向（P1，可选落仓）

- **E2E**：在「多表面/仅默认表面渲染」类产品形态下，断言 **可见 DOM/画布** 与 **权威快照** 一致（须冻结数据与路由占位）。
- **契约测试**：对「嵌套结构化字段」断言 **类型与形态**（例：禁止把 object 误存为单层 JSON 字符串）；流式场景对 **聚合后的最终载荷** 断言。
