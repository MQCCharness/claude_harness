## 1.1 技能正文通用化（横切 · 与具体应用解耦）

蜂群配套技能默认服务**多仓库、多产品**的协作；正文应 **模式级**、**可迁移**，避免把某一子项目写成唯一真理来源。

1. **禁止**在技能正文中以**单一应用仓库名、内部代号、专有工具名、唯一 REST 路径**作为**唯一**教学案例（除非该技能声明为「某仓专用附录」并单独索引）。
2. **反模式 / 正例**优先用 **短标签 + 尖括号占位**（例：`反模式：嵌套字段字符串化（<yourField>）`），见 **`17`** §2.1 风格。
3. **可验证命令**用 **参数化**（`{REPO_ROOT}`、`{PKG_MANAGER}`）或写「以本工作区已入仓脚本为准」，避免复制粘贴即失效的盘符路径。
4. **产品专项复盘**（某次发布的根因、某客户环境）优先 **`.claude/swarm/RECENT-DECISIONS.md` 或产品仓文档**；技能只抽象出 **可复用的排查分层与门禁**。
5. **与本工作区绑定且合法的锚点**：CLAUDE.md 路径、技能文件名、`CHAIN/ROUND` 格式、**本仓库技能树全局索引**—属于**元架构**，不视为「单一应用特化」。

## 2. 常见误读与正读

| 误读 | 正读 |
|------|------|
| 「每条消息都要满编四轮 Agent 子代理」 | 「每条消息」指统帅按 **4+1 总线纪律**工作；Agent 子代理仅在 **显式多角色 / 写码回合 R3+R4 / R2 命中 / 用户满编** 等情形下强制。 |
| 「混写触发与任务同条 = 自动满编五段」 | **否**。蜂群 CLAUDE.md **解析与优先级（模式状态先于子任务）与混写**规定：**优先**指先固定模式状态与强制 Read/闸门，**不**把混写升格为满编；满编仍仅由 **满编口令**、军团 **`叶 OP 与末端 L2 蜂群`** 专条 等已定条件决定。 |
| 「最小充分 = 永远不必 R2」 | **R2 强制触发**命中时仍须 R2 或 **一句边界说明**；**领域不熟升格**与裁量并存，以 CLAUDE.md 与 `11` 为准。 |
| 「最小充分 = 可省写码回合 / 满编 / 叶 OP 五段 / R4 实测」 | **否**。**最小充分**不得架空 **写码回合**、**满编口令**、**叶 OP 末端**、**时序闸门**、**R4 实测 / R4b**；见 CLAUDE.md **反降级总闸**（蜂群 + 军团互指）。 |
| 「统帅可以顺手把业务代码/产品长文也在主对话写完」 | **否**。默认 **R3 Agent 子代理**；主对话直改 **业务树** 须 **`主帅改业务仓`/`主帅落地业务代码`**（同义可接受）且 **首段 RISK_OWNER**；治理白名单见 CLAUDE.md **反降级总闸**。 |
| 「R3+R4 并行 = R4 可以先读仓库」 | **已废止**：须遵守 CLAUDE.md **蜂群编队时序闸门**——**R4a** 与 R3 可并行但**仅限契约级**；**R4b** 仅在 R3 关门与证据回灌后。 |
| 「诚实边界 = 可以不开 Agent 子代理」 | 仅 **客观无法** Agent 子代理时启用；禁止用其规避写码回合或显式多角色隔离。 |
| 「技能 11 = 每条对话先 Phase A」 | Phase A/B 仅在 **升格链**（不熟领域/用户模板启动/统帅判定）下启用。 |
| 「mock 全绿 = LLM 路径已验收」 | **集成层 P0** 还须 **录制夹具 / 可开关观测 / 真实环境冒烟** 之一，见 CLAUDE.md「第三方 API...」与 **`13`**。 |
| 「子代理说已实现 = 可收束」 | 统帅 **须 diff+test** 核验，见 CLAUDE.md 与本文件交叉引用 **`13` §5**。 |
| 「蜂群汇流 = 全部结论已验证」 | 主帅须标 **A/B/C**；**B/C 不可冒充已拍板**；见 CLAUDE.md「子代理结论可验证分级」与 **`14`**。 |
| 「子代理下一轮自动记得终端输出」 | **否**；须 **证据回灌**（Agent 子代理内摘要或落盘路径），见 CLAUDE.md「Sub-agent 可观测性...」与 **`14` §4**。 |
| 「A/B/C = 升格 Phase A/B」 | **否**。**可验证级**称 **A/B/C**；需求升格称 **Phase A / Phase B**（技能 `11`），汇流勿混用简称。 |
| 「命中扩展触发词 = 必须满编」 | **否**。CLAUDE.md 与 **`16`** 明确 **分层触发**：话术命中加强总线与形状保持；**是否 Agent 子代理** 仍按强制情形与最小充分集裁决。 |
| 「某次工具调用成功 = 界面必然可见变化」 | **否**。须区分 **权威状态快照是否已变** 与 **当前视口/渲染规则是否呈现**；先按 **`17`** 分层排查，再定责上游或客户端。 |
| 「形状保持 = 永远不能改需求」 | **否**。用户当轮**明示**改需求或新任务时以当轮为准；须**首段声明**哪些冻结条作废。 |

| 「留痕三档的 L0/L1 = 技能 16 分层触发的 L0/L1」 | **否**。**留痕三档**（叙事稿 / 结构化 / 字节镜像）与 **`16`** 的 L0–L3 **话术分层触发**不同名；勿混用代号。 |
| 「CHAIN/ROUND 可替代 A/B/C 可验证级」 | **否**。**CHAIN** 只管任务链隔离与排序锚点；**A/B/C** 仍表示结论可验证程度。 |

| 「人设可以要求 AI 当律师/医生下结论」 | **否**。须持证角色或法定背书的，统帅须**拒绝或降级**为信息整理 + 建议咨询持证方，除非用户明示豁免且风险自担（仍受法律与平台底线约束）。 |
| 「说了强制身份开场仍不先贴三问」 | **否**。命中口令则**下一条助手回复**须先 **§3.2** 再办其他事（同条消息已答全三字段时可摘要确认替代重复提问）。 |

## 3. R1/R2 → R3/R4 转化表（可复制模板）

汇流时推荐最少列：

| 来源 | 结论摘要 | 落点 | 下一轮动作 |
|------|----------|------|------------|
| R1 / R2 | 一句话 | →R3 / →R4 / 风险 / 待决 | 发 Agent 子代理 / 更新冻结表 / 阻塞用户 |

**填写纪律**：每条上游结论**至少一行**；禁止用「见上文」代替落点；合并组须在摘要中列全要点。

### 3.1 统帅汇流输出骨架（迷你模板 · 与 CLAUDE.md 对齐）

轮末向用户汇流时，建议按顺序压缩呈现（详略由统帅裁量，**语义块不得无故对调**）：

0. **身份与意图三字段**（执行者身份真实/假定 · 期望扮演身份/人设 · 产出下游用途；各标 **A/B/C** 或待决）
1. **其余冻结事实表**（A/B/C 已标注）
2. **R1/R2→R3/R4 转化表**（或「本轮无上游转化」）
3. **与用户历史结论的矛盾**（沿用 / 推翻 / 待决）
4. **诚实边界**（Agent 子代理/证据/环境限制）
5. **待决与指针**（路径、`CHAIN`/`ROUND`、下一轮问谁）

### 3.2 身份与意图三问 · 强制开场模板与快速选项（与 CLAUDE.md「强制身份开场」口令对齐）

**何时用**：用户消息含 `强制身份开场` / `身份开场` / `三问开场` / `先三问` / `身份三问` 之一时，统帅**下一条回复必须先贴本节**，再处理同条消息里的其他任务。

**三问（用户可自由答，或从下方快速选项复制改半行）**

1. **执行者身份**（真实或假定）：你是谁、以什么身份在发起本轮协作？（假定身份请写明「假定：...」，**非**实名认证。）
2. **期望扮演身份（人设）**：你希望我以什么专业边界与口吻回应？（须与蜂群底线兼容，无证照法定背书。）
3. **产出下游用途**：本轮产出你主要会**拿去做什么**？（自学 / 入仓 / 对内汇报 / 对外材料 / 招投标或审计附件 / 自动化输入 / 其他——请具体到场景。）

**快速选项（整行复制回贴即可，可改括号内文字）**

- **执行者身份**：`【执行者身份】真实·本人账号自用` / `【执行者身份】假定·我是某公司产品经理（虚构）` / `【执行者身份】假定·我在做课堂作业角色扮演（非真实执业）`
- **期望扮演身份**：`【人设】中立助手，偏结构化清单` / `【人设】行业顾问口吻，仅信息整理与可核验引用，不给法定结论` / `【人设】按本仓库蜂群 R3 写实现草案，须标回滚点`
- **产出下游用途**：`【用途】仅当轮对话自学，不外发` / `【用途】写入本仓库代码/配置并跑 CI` / `【用途】对内汇报材料（脱敏）` / `【用途】对外客户可读文档（须合规复核）` / `【用途】招投标/审计附件（须与招标文件或审计清单对齐）`

**统帅紧跟一句**：「以上三字段我将标为 A/B/C：...（或待你确认后标 B→A）」再进入正题。

## 4. 冻结验收表 schema（最小）

与 CLAUDE.md 中 **R4 清单执行门禁** 对齐，建议列：

| 字段 | 说明 |
|------|------|
| **TC-xxx** | 稳定编号；**业务/方案类**建议前缀 **`TC-B-`**，**工程/构建类**建议 **`TC-E-`**（与 CLAUDE.md「R4 验收分轨」对齐） |
| **OMIT_REASON** | 可选；当结构化留痕省略内容时填 `CODE_BLOCK` / `TOOL_PAYLOAD` / `SYSTEM_INJECT` / `DUPLICATE_TECHNICAL` 之一，并一句替代验证手段 |
| **标题** | 可理解的验收点 |
| **步骤/命令** | 可复现（含工作目录若有关） |
| **期望** | 退出码、关键输出或行为 |
| **优先级** | P0 / P1 |
| **结果** | 通过 / 失败 / 跳过+原因 |
| **备注** | 环境、豁免、链接到风险登记 |

**裁决**：统帅在交付中给出 **PASS / COND_PASS / FAIL** 定义见 CLAUDE.md；**禁止**「理论上应过」替代实测（豁免路径除外）。

## 6.1 路径书写规范（移植友好）

**目的**：规则、技能与落盘文档中的路径可被任意机器克隆后直接使用，不因盘符或本机目录名失效。

| 约定 | 说明 |
|------|------|
| **仓库内资源** | 一律写**相对仓库根**的路径，推荐 **POSIX 正斜杠**：如 `.claude/rules/CLAUDE.md §二`、`.claude/skills/hajimi-dog/skills_linter.py`。 |
