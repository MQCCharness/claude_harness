# 领域不熟与需求升格蜂群协议

本技能是工作区规则 **「领域不熟与需求升格」** 的操作细则：**不新增固定角色**，仍由 **统帅** 编排，**R2 / R3 / R1** 分轨承担「情报—方案—攻击面」，**R4** 仍在契约冻结后按黑盒纪律产出验收清单，**统帅**负责实测门禁与 `PASS` / `COND_PASS` / `FAIL` 裁决（见 `09_任务驱动型四角色对抗开发工作流.md`）。

## 1. 问题定义（何时启用）

- **典型信号**：用户承认不熟某赛道；说不清「别人默认有什么」；功能像清单但缺少**对标**与**成功判据**；多解释并存且缺少可核验来源。
- **本协议解决什么**：把**选项空间**与**行业地面**摊开，再让用户做取舍；**不代替**用户做商业终局决策。
- **本协议不解决什么**：不能在没有证据链时承诺「全球最先进」；不能把「执行精准」自动变成「战略正确」。

## 2. 与「一页纸 / 撕裂回合」的衔接

| 片段 | 作用 |
|------|------|
| **一页纸（契约摘要）** | 背景、用户与场景、目标、**非目标**、成功标准（可测）、一期范围、假设与待证实项。 |
| **撕裂回合（R1）** | 攻击矛盾、隐含假设、灾难路径与安全刃口；本协议中的 **R1 竞争力轮**侧重「输给主流的叙事」。 |
| **本协议 Phase A/B** | **先 R2 对齐外部世界**，再 **R1 撕裂**，避免闭卷自嗨；顺序可因任务压缩，但**不得跳过 Phase A 却声称已完成对标**。 |

## 3. Phase A — R2 行业与事实基准（Agent 子代理）

**目标**：得到可讨论的「地面」——**table stakes**（不做会显得落后）、常见**差异化轴**、候选参照对象与**待核验**清单。

**R2 Agent 子代理建议输入（最小充分）**：

- 产品一句：为谁、在什么场景解决什么问题。
- 已知约束：平台、地区/合规敏感点、工期与团队规模（若可知）。
- 用户已列功能（若有）：仅作「初稿」，不要求正确。
- 输出结构要求（强制 R2 按表/条列回复）：

  1. **参照系矩阵**：3～5 个候选产品/开源/标准（说明为何同赛道）。
  2. **默认期待（table stakes）**：列表 + 每条标注 **已核实 / 推断 / 待核验** 与证据类型（官网、Release notes、文档条款、实测、二手综述）。
  3. **差异化轴**：体验、性能、垂直场景、集成、价格模型、合规、工作流深度等中，本赛道常见切分。
  4. **术语与坑**：易混概念、许可证或平台限制线索（若有）。

**退出条件**：统帅判断已足以支撑 Phase B 的「标准档」讨论；否则 **待核验** 项进入 **待用户决策 / 下一轮 R2**，未闭合则**降级**「标准档」承诺范围。

**统帅义务**：将 Phase A 落入 **R1/R2 产出转化**（→R3 硬约束、→R4 契约与 P0/P1、风险、待决），**禁止**只把 R2 报告贴在对话末尾。

## 4. Phase B — MVP / 标准 / 野心 + R3 性价比（Agent 子代理）

在 Phase A 汇流稿基础上，由 **R3 Agent 子代理**（可并行由统帅先给摘要）产出三档草案：

| 档位 | 含义（可本地化命名） |
|------|----------------------|
| **MVP** | 最小可验证价值，明确**不承诺**的行业默认能力。 |
| **标准** | 与 Phase A 中「table stakes」基本对齐的体面交付。 |
| **野心** | 在约束允许下的超额能力，标出**依赖与风险**。 |

**R3 须标注**：各档的工程量级、技术依赖、主要风险、回滚点、**性价比最高的一条差异化**（若与用户目标冲突须写明）。

**用户/统帅拍板**：默认交付档、**非目标（至少三条）**、分期路线（若采纳）。未拍板项写入 **待用户决策**，**禁止**默示替用户选「野心档」。

## 5. R1 竞争力与威胁面（可选专轮，Agent 子代理）

在已有 A/B 草案后发起 **R1 Agent 子代理**，侧重：

- 若只做**用户最初功能清单**，在哪些对比维度上会输给 Phase A 中的参照系？
- **虚假差异化**、不可测指标、隐私/合规/滥用面、运营与极端失败路径。

产出须可被统帅转为 **→R4** 的负例与 **P0** 候选（及已知风险登记）。

## 6. 统帅门禁（进入写码前）

- **已冻结**：目标用户与场景、选定档位（或分期）、关键可测成功标准、**非目标**、关键 **待核验** 项的责任人与时限（若仍开放须标风险）。
- **转化表已更新**：R1/R2 要点已落入 R3/R4/风险/待决（与规则「反孤岛」一致）。
- **然后**才进入常规 **写码回合**（R3+R4 Agent 子代理底线 + R4 清单实测门禁），除非用户明示豁免相关条款。

## 7. 用户可复制 Prompt 模板

**模板 A — 启动 Phase A（R2）**

> 我对该细分领域不熟，请先不要按我最初列的功能直接实现。请发起 **R2 Agent 子代理** 做 Phase A：**竞品与行业基准简报**（参照系矩阵、table stakes、差异化轴），每条标注 **已核实 / 推断 / 待核验** 与证据类型；不要无引用自称 SOTA。已知：产品一句 ___；约束 ___；我草稿功能 ___（可错）。

**模板 B — 启动 Phase B（三档 + R3）**

> 在 Phase A 汇流稿基础上，请发起 **R3 Agent 子代理** 输出 **MVP / 标准 / 野心** 三档及各自成本·风险·回滚；标出**性价比最高的一条差异化**。默认档待我确认；请列出**非目标**至少三条。

**模板 C — R1 竞争力攻击**

> 请发起 **R1 Agent 子代理**：若只做我最初功能清单，相对 Phase A 参照系会在哪些维度失败？最坏用户叙事是什么？哪些点必须进 **P0 验收** 或登记为已知风险？

## 8. 诚实边界（强制）

- **禁止**：在无检索、无可引用来源时输出「全球最先进」「业界第一」「唯一正确方案」等无法证伪的断言。
- **必须**：事实主张带 **置信度** 与 **证据类型**；**待核验** 不得伪装成已核实。
- 与规则 **R2 事实绑定** 一致：可核验结论须能落成 **R3 可执行约束** 或 **R4 可测契约**，而不是停在「建议自行搜索」。

## 9. 常见误读（对齐工作区规则）

- **非**「每条消息必须满编四轮 Agent 子代理」：仍由统帅 **最小充分角色集**裁量；用户要求满编时才四轮齐发。
- **非**「可默省略 R2」：命中 **R2 强制触发**（含领域基准不清）须 **R2 Agent 子代理** 或统帅 **一句说明为何本轮不需要**。
- **写码回合**改动规则与本技能等仍属 **R3+R4** 底线（除非用户明示豁免）。
- **R4** 仍默认 **黑盒契约级输入**；不要把「读某路径源码」默认塞进 R4 Agent 子代理（除非用户明示白盒并标注例外）。

## 10. 与 CLAUDE.md 优先级及「最小充分」的关系

- 本技能的 Phase A/B **仅在**统帅判定为 **领域不熟 / 需求升格** 或用户按 **§7 模板**主动启动时走满；**不要求**每条普通对话都先跑对标。
- 与 **写码回合**、**投机实现**标注、**R3+R4** 的关系以 CLAUDE.md §二 **「领域不熟与需求升格」** 整节为准；用户强推未冻结契约的实现时，**不得**静默免除 **R3+R4**（除非用户同时明示豁免）。
