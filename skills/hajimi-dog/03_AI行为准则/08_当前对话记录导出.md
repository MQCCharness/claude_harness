# 蜂群会话留痕与导出（三档 · 与「不省略」对齐）

**权威**：强制条款以 **CLAUDE.md §二** 为准；本文件是**操作说明与命名约定**，不得收窄写码回合、R4 实测等底线。

## 1. 留痕三档（勿与技能 `16` 的 L0–L3 触发分层混名）

| 档位 | 用途 | 典型载体 |
|------|------|----------|
| **叙事稿** | 人读、讨论、对外脱敏摘要 | 主帅汇流正文、自然语言导出 MD |
| **结构化全记录** | 验收、grep、跨会话接力 | 冻结事实表、**TC-B-** / **TC-E-**、转化表、命令与退出码摘要、`.claude/swarm/` 简报 |
| **字节级镜像** | 审计对账、diff | 行式 transcript 导出、脱敏日志/报文；**禁止**密钥与完整隐私请求体 |

**原则**：叙事稿**不构成** **A 级**可验证结论（见技能 **`14`**）。

## 2. 「不能省略」与 OMIT_REASON

当可读性、脱敏或体积要求与「逐字不删」冲突时，**禁止静默省略**：在**结构化全记录**中写明 **`OMIT_REASON`**（封闭枚举，见 CLAUDE.md）并一句说明替代验证手段（TC 编号 / 落盘路径 / 哈希等）。

枚举：**`CODE_BLOCK`**、**`TOOL_PAYLOAD`**、**`SYSTEM_INJECT`**、**`DUPLICATE_TECHNICAL`**。

## 3. 导出是否收录「身份与意图三字段」

- **叙事稿**：可收录统帅已冻结的三字段摘要（脱敏后），便于复盘「当时认为谁在请求、人设、用途」。
- **结构化全记录 / 字节镜像**：若含用户自述身份或用途，须遵守**隐私最小化**与 **OMIT_REASON**；**禁止**为留痕而收集不必要 PII。

## 4. Agent 子代理对账（锚点 · 非 mtime）

- 满编或显式 R1–R4 Agent 子代理时，正文**第一行**须符合 CLAUDE.md **首行锚点**（`CHAIN` / `ROUND` / `ROLE` / `PARENT_REF`）。
- 复盘时**勿用**子代理文件的修改时间当**因果序**；以 **ROUND**、轮次表或主会话条目序为准。

## 5. 本仓库示例脚本（可重复工具链）

工作区示例：`./_export_sales_swarm_full_md.py`（相对仓库根）

- **默认**：生成**自然语言稿**（仅叙述性 text，清洗围栏代码与技术字面量）。
- **`--full-technical`**：生成**技术全量稿**（结构化 JSON 片段 + 原始行式记录镜像），用于审计对账。

运行（在**仓库根**）：`python _export_sales_swarm_full_md.py`；须指定会话目录时使用 `--transcript-base <目录>` 或环境变量 `SWARM_TRANSCRIPT_BASE`（详见脚本 `--help`）。
