# HajimiDog 动态上下文快照 (Dynamic Context Snapshots)

本文件用于存储和管理 HajimiDog 项目在各个重要里程碑节点的"状态快照"。
**目的**：避免 AI 在每次对话时从零开始扫描整个项目，通过读取快照瞬间恢复对当前项目架构和核心变量的记忆，节省 Token 并降低遗忘率。

## 快照生成与加载规则
1.  **生成时机**：当完成一个独立的功能模块（如：蓝牙联调通过、OLED重构完成）并得到用户确认后，AI 必须主动生成一个新的快照并追加到本文件中。
2.  **加载时机**：在新的对话窗口开始时，如果用户要求继续开发，AI 应优先读取本文件中的最新快照，而不是去盲目扫描 `main.c` 或前端源码。

## 存档点 1：初始架构成型 (Initial Architecture Setup)
*   **时间戳**: 2026-03-20
*   **当前状态描述**: 项目分为嵌入式固件端 (C语言, 裸机) 和 可视化上位机端 (React+Vite)。
*   **嵌入式核心状态**:
    *   舵机 PWM: PA0~PA3, PA6 (TIM2/TIM3)
    *   LED PWM: PB0, PB1 (TIM3)
    *   OLED I2C: PB8(SCL), PB9(SDA)
    *   蓝牙 UART3: PB10(TX), PB11(RX)
    *   逻辑流向: `main.c` 依赖 `Action_Mode` 进行非阻塞调度。蓝牙中断负责修改 `Action_Mode` 和 `Face_Mode`。
*   **前端核心状态**:
    *   已支持节点: 基础触发器 (蓝牙接收)、基础动作 (切换状态、更新OLED)。
    *   代码生成引擎: 基于 Zustand 状态树遍历，生成基础的 `if-else` 状态机代码。
*   **待办/下一步**: 暂无未完成的硬性 Bug。准备进行具体的模块功能扩展或前端 UI 优化。
