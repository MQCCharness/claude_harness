# 外部 Cursor 技能包 · 桥接索引（机制对齐，不合并领域正文）

## 1. 目标

- **保留** 现有 **角色（R1–R4 + 统帅）**、**CLAUDE.md 强制条款**、**全局索引 + 热度链条** 的触发效率。
- **增加** 与 **外来第三方技能包**（对方仓库内层可能仍名 `cursor_skills/`，属**他方结构名**）之间的 **显式路由**，便于在**正确任务**下借鉴其 **INDEX / 触发词 / 关联度 / 工作流** 做法。
- **禁止**：把外来包中**异工程领域**文档批量迁入本仓库 **`.claude/skills/`** 树；**禁止**将外来解压目录直接当 **Claude Code 项目技能根** 使用。

**外来根路径（导入暂存区 · 默认约定 · 相对仓库根）**：`_import/external-cursor-skills/`（旧约定 `_import_cursor_skills/cursor_skills/` 若仍在用，在本文件与 `external-cursor-pack-bridge/SKILL.md` **同步一行**实际路径即可。）
**桥接 SKILL（窄描述、防误触发）**：`.claude/skills/external-cursor-pack-bridge/SKILL.md`

**GitLab 可读副本**：`docs/claude-governance/external-pack-bridge/15-外部技能包桥接索引.md`

## 3. 与 4+1 角色的协作方式（借机制不借壳）

| 角色 | 外来包的合理使用 |
|------|------------------|
| **+1 统帅** | 决定「本轮是否触碰外来路径」；遵守 **解析→分发→汇流→续聊**；外来内容只作**分包摘要**进 Agent 子代理或落盘指针。 |
| **R1** | 可对照外来「虚拟任务预演」检查**是否漏查索引**；不引入异工程威胁面当本仓库默认假设。 |
| **R2** | 外来 `_meta`、官方式 workflow 适合作为 **事实与引用结构** 的参考；事实进汇流须 **A/B/C**。 |
| **R3** | 可借鉴「双代码树对齐」等**通用流程**；实现约束仍以本仓库代码与 CLAUDE.md 为准。 |
| **R4** | 可借鉴「操作前查阅 / 验收条目化」风格；**清单执行与 PASS/FAIL** 仍在统帅侧，对齐 CLAUDE.md R4 门禁。 |

## 4. 推荐调用顺序（外来相关任务）

1. 若任务明确涉及外来包：先 Read **`external-cursor-pack-bridge/SKILL.md`**（本文件上一节根路径）。
2. 路由：外来 **`INDEX.md`** → 再读 **单篇** workflow 或领域文，**禁止**同轮通读 `_meta` 详版全家桶。
3. 回到本仓库：用 **`00_全局索引`** 定位技能；蜂群裁决仍以 **CLAUDE.md + 09/11/12/13/14** 为准。
4. 需跨会话继承：用 **`14_蜂群可验证结论与跨会话裁决落盘`**，对外来结论同样标 **A/B/C**。

## 5. 维护约定

- 外来包版本升级：只需更新 **桥接 SKILL** 与 **本节「外来根路径」** 一句；**不必** diff 合并领域 `.md`。
- 本仓库技能增删：仍跑 `skills_linter.py`，并在 **全局索引** 补链条；可在本文件「机制对表」增一行**对位说明**，勿粘贴大段外来原文。
