# 军团模式：回声校验与规模裁量

**范围**：军团模式阶段六（回声校验）的操作细则，以及 S/M/L 规模判定的裁量指南。

## 0. Phase Gate（阶段门禁）

### 0.1 定位

Phase Gate 是 DAG 驱动执行中**每个 EP 之间**的局部质量关卡，与 §1 回声校验（全局终态检查）形成**双层校验**。Gate 不修改、不覆盖`CLAUDE.md §二` 的 **R4a 清单 + 统帅实测 + R4b 关门** 门禁。

### 0.2 Gate 检查清单模板（按层级分型）

Gate 在通用四项检查基础上，追加当前 EP 涉及层级的**专属检查**。

**通用四项**（所有 Gate 必做）：G1 全部角色已返回 | G2 I/O 契约校验（格式+必需字段+decisions_log）| G3 无未解决矛盾 | G4 军令状穿透完整性抽查

**EX 层专属**（本 EP 含 EX 时追加）：GE1 纲要覆盖军令状全部冻结目标 | GE2 任务书拆解完备（下级均有对应） | GE3 跨域协调点已标注 | GE4 EX 追加约束与军令状兼容

**MG 层专属**（本 EP 含 MG 时追加）：GM1 执行计划与 EX 纲要对齐 | GM2 OP 任务书可执行且 I/O 契约完备 | GM3 MG 追加约束与上游兼容 | GM4 资源/时序分配合理

**OP 层专属**（本 EP 含 OP 时追加）：GO1 产出符合 I/O 契约格式 | GO2 蜂群验收结果（**PASS / COND_PASS / FAIL**；`COND_PASS` 须含 **COND_ID**、条件、负责方、时限，与`CLAUDE.md §二` 定义一致）| GO3 decisions_log 完整 | GO4 产出在任务书范围内（无越界）

**逐级汇流专属**（本 EP 任一返回含 L3 汇流义务时追加）：GR1 产出含 **《逐级主题汇流块》** 且字段非空 | GR2 **直连下级清单**与冻结架构一致，状态与事实可对账 | GR3 **下级到齐状态**为 `齐` 或已标注 `COND`/`缺_*` 且未瞒报 | GR4 若宣称已下发下级 Task，**首行锚点或统帅代发留痕**存在（禁止「未创建却标 A 级已派发」）

**裁决**：**PASS** / **BLOCK** / **COND_PASS**（附条件放行）。历史写法 **CONDITIONAL_PASS** 与 **COND_PASS** 同义；宣告 **COND_PASS** 须登记 **COND_ID**、条件说明、负责消化方、消化时限、关联 **TC-ID**（若已冻结），与`CLAUDE.md §三` Phase Gate 专节一致。

### 0.2a COND_PASS 与蜂群门禁对齐（必读）

- 军团 **Phase Gate** 与 **回声**中的 **COND_PASS** **不得**弱于`CLAUDE.md §二`「R4 清单执行门禁」：**全部 P0 实测通过**前提下，方可因 **P1 未自动化** 或**已文档化剩余风险**宣告 **COND_PASS**。
- 蜂群 **`FAIL`** → 军团 **不得**宣告全局 **PASS**；叶 OP Gate **不得放行**（见`CLAUDE.md §三` 叶 OP 层 Gate）。

### 0.2b 回声 ↔ TC 双向锚定

- 回声表或 `ECHO-CHECK.md` 中凡声称 **可验收 / P0 已满足 / 对齐军令状⑤**，须有 **`TC-B-*` / `TC-E-*`** 互指（或「待补 TC-ID + 责任方 + 时限」）；**主观满意**须 **`ACCEPT` + TC-ID** 方可记 **A 级**。
- 末端叶 OP 的 **R4a 冻结表**为优先事实源；回声行 **禁止**与 R4a 表两套叙述无互指。

### 0.2c 收敛与轮次熔断（与`CLAUDE.md §三` 一致）

- **收敛**：满足 **A 级 TC 闭环** 或 **PASS** 或合法 **COND_PASS** 或 **`ACCEPT` 留痕**（见`CLAUDE.md §三`「收敛、轮次熔断与 A 级核验」）；未收敛须继续改与验。
- **叶 OP**：同一目标连续 **5** 次 **R3→R4b** 仍不收敛 → **强制重入 R1+R2**（蜂群时序闸门），再 **`R3+R4a+R4b`**。
- **军团**：同一 **CHAIN** 下回声连续 **5** 次仍 **BLOCK** / 未合法 **COND_PASS** → **强制 R1 型 + R2 型 Task**（或用户同意改军令状/DAG）后重跑。

### 0.2d 证据权威序（Gate / 回声 / 蜂群 TC · 与 G/L/O 对齐）

- **叶 OP 蜂群**：**R4a 表号 + +1 统帅实测** 优先于 **R4b 口头 PASS**；**R4b** 与 **统帅命令输出** 冲突时，以`CLAUDE.md §二`：**仓库与命令输出** 为准。  
- **回声表** 宣称对齐军令状⑤ 须有 **TC-ID** 互指；**G3 矛盾未解** 时 **不得**用 **COND_PASS** 消化（与 §0.2a 并列理解）。  
- **事实矛盾** 触及军令状 / `[PENETRATING]` 时须走军团 **G** 或 **L**（见 `CLAUDE.md §三`），**不得**仅在下级蜂群内 **O** 式修补糊弄过关。  
- **S/M/L 规模判定** 本身为 **操作层叙述**，**禁止**冒充 **A 级**全局事实除非有可复核采样。

### 0.3 失败处理梯度

```
BLOCK
 ├─ 第 1 次：统帅按失败类型处理
 │     ├─ G1 失败（未返回）→ 等待/resume/重发 Task
 │     ├─ G2 失败（I/O 不匹配）→ 统帅转译 或要求上游重做
 │     ├─ G3 失败（矛盾）→ 统帅裁决 或请用户拍板
 │     └─ G4 失败（穿透被篡改）→ 严重违规，要求重做
 ├─ 第 2 次：须向用户汇报并提供选项
 │     ├─ ① 用户介入裁决
 │     ├─ ② 降级该角色（统帅代理）
 │     └─ ③ 回退到阶段三调整架构
 └─ 禁止：BLOCK 未解决就跳过 Gate 继续执行
```

### 0.4 与降级的整合（阶段压缩）

当 Task 资源不足时，EP 可压缩：

| 压缩方式 | 条件 | Gate 处理 |
|---------|------|---------|
| 同 EP 分批（>4 角色） | 角色数 > 并发限制 | 批间加微型 Gate（仅检 G1+G2） |
| 相邻 EP 合并 | Task 总预算不足 | Super-EP 结束后执行完整 Gate |
| 线性退化 | 仅 1-2 Task | 每步即检（G1+G2 简化版） |
| 诚实边界 | 无 Task | 统帅分节扮演，每节末自检 I/O 格式 |

### 0.5 与蜂群 R4 门禁的层级关系

| 门禁层 | 职责 | 检查范围 |
|--------|------|---------|
| **军团 Phase Gate（L3）** | 跨角色集成质量 | 产出完备、格式合规、可被下游消费 |
| **蜂群 R4 门禁（L2）** | 角色内部质量 | **R4a** 黑盒清单 + 统帅实测 + **R4b** 关门；**PASS / COND_PASS / FAIL** |

蜂群 **FAIL** → 该叶 OP 产出不达标 → 军团 Phase Gate / 回声 **不得**覆盖为 **PASS**；**COND_PASS** 须与蜂群侧 **COND_ID** 留痕互指。蜂群 **COND_PASS** → 军团 **不得**仅因「非 PASS」误杀，须核对`CLAUDE.md §二` 条件。

## 1. 回声校验执行清单

### 1.1 触发时机

军团统帅在**所有末端蜂群产出已回传、中间层已汇总**之后、**向用户输出最终结论之前**执行。

### 1.2 校验模板（可复制）

```markdown
## 回声校验记录

**任务链 ID**：___
**校验时间**：___

### 逐条比对

| # | 军令状要素 | 原文摘要 | 对齐结论（PASS/BLOCK/COND_PASS） | TC-ID / COND_ID | 偏离或条件说明 |
|---|-----------|---------|------------------|----------------|----------------|
| 1 | ② 冻结目标句 | … | PASS / BLOCK / COND_PASS | TC-… 或 NONE | … |
| 2 | ③ 硬约束-条目1 | … | … | … | … |
| … | … | … | … | … | … |
| N+1 | ⑤ 成功标准-条目1 | … | … | **须**有 TC-B-* 或 COND_ID | … |

### 跨部门一致性检查

| 部门A产出 | 部门B产出 | 是否矛盾 | 说明 |
|----------|----------|---------|------|
| … | … | ✅ 一致 / ⚠️ 矛盾 | … |

### 校验结论

- **`PASS`**：无 **BLOCK** 项，且 **P0 级 TC** 均有 **A 级**证据或合法 **COND_PASS** 留痕
- **`BLOCK`**：存在未解决偏离 → 列出项，要求重做或升级裁决；**禁止**直接终稿
- **`COND_PASS`**：与`CLAUDE.md §二` 一致；须登记 **COND_ID**、条件、负责方、时限
- **主观项**：须 **`ACCEPT` 记录** + **TC-ID** 方可视同 **A 级**
```

### 1.3 偏离处理流程

```
发现偏离
    ↓
军团统帅标记 **`BLOCK`（回声失真）**
    ↓
判断偏离性质：
  ├─ 目标偏离 → 要求相关部门重新执行
  ├─ 约束违反 → 强制重做，不可接受
  ├─ 非目标越界 → 裁剪越界部分或请用户确认扩展
  ├─ 标准未满足 → 补充执行或降低预期（需用户确认）
  └─ 跨部门矛盾 → 军团统帅裁决，或请用户拍板
```

**禁止**：发现偏离后不处理直接输出；把偏离标为「可接受」却不告知用户。

## 2. S/M/L 规模裁量指南

### 2.1 判定维度

统帅从以下维度综合评估：

| 维度 | S 特征 | M 特征 | L 特征 |
|------|--------|--------|--------|
| **领域跨度** | 单一领域内 | 单一领域有深度 | 跨 2+ 专业领域 |
| **交付物数量** | 1 个 | 2-3 个相关联 | 4+ 个或多类型 |
| **技术栈** | 单一技术 | 主技术+辅助 | 多技术栈并行 |
| **协作角色** | 不需要对抗 | 需多视角验证 | 需多部门分工 |
| **风险面** | 低 | 中，可控 | 高，跨域耦合 |
| **预期工期** | 分钟级 | 小时级 | 天级或以上 |

### 2.2 判定输出模板

统帅在首轮回复中须包含：

```markdown
**规模判定**：[S / M / L]
**理由**：[一句话说明为何是该等级]
**启动架构**：[L1 直执行 / L2 蜂群 / L3 军团]
> 如需调整，请告诉我升级或降级。
```

### 2.3 边界情况

| 情况 | 处理 |
|------|------|
| M 但用户要求军团 | 尊重用户，升级为 L |
| L 但用户要求只用蜂群 | 尊重用户，降级为 M；统帅一句提醒可能的覆盖不足 |
| 初始判 M，执行中发现跨域 | 统帅建议升级为 L，用户确认后切入军团模式 |
| 初始判 L，执行中发现范围收敛 | 统帅建议降级，将已完成部门的产出作为蜂群输入继续 |

## 3. 军团规模控制策略

### 3.1 最小充分部门集

与蜂群「最小充分角色集」对齐：

- **核心部门**：直接服务于冻结目标的部门，**必须**启动
- **辅助部门**：间接支持的部门，**按需**启动
- **可选部门**：锦上添花的部门，用户要求或高风险时启动

统帅在输出架构方案图时，须标注每个部门属于 **核心/辅助/可选**。

### 3.2 按需展开（渐进式）

当对资源或成本有顾虑时，推荐渐进式展开：

```
第一波：只启动核心部门 + 末端蜂群
        ↓
第一波产出汇流 + 回声校验
        ↓
判断：辅助部门是否必要？
  ├─ 是 → 第二波启动辅助部门
  └─ 否 → 直接进入最终汇流
```

### 3.3 降级策略（Task 不够用时 · 按三层级细化）

当平台 Task 并发/总数不足时，按以下优先级**从底部保留**：

| 优先级 | 保留对象 | 层级 | 降级方式 |
|--------|---------|------|---------|
| **P0（不可降级）** | OP 蜂群 **R3+R4a+R4b**（写码链按时序闸门） | OP/L2 | — |
| **P1** | OP 蜂群完整 4+1 | OP/L2 | R1/R2 降为统帅分节 |
| **P2** | OP 角色 Task | OP | 同部门 OP 合并串行 |
| **P3** | MG 角色 Task | MG | MG 由统帅分节代理 |
| **P4** | EX 角色 Task | EX | EX 由统帅分节代理 |
| **P5（最低）** | DAG 顺序与 I/O 格式 | 全局 | 统帅分节按拓扑序+I/O 格式 |

**降级逻辑**：从 P4 开始降；P0 不可触碰。

**降级声明**（统帅首段强制）：说明哪些 EX/MG 被降级为分节代理，OP 保持 Task 隔离，降级不影响军令状穿透、I/O 格式、Gate 检查。

**分节代理纪律**：统帅代理 EX/MG 时，须按角色**独立分节**输出（标注 `[代理-EX]` 或 `[代理-MG]` + 角色名），**禁止**同一段落内混扮多个角色。

## 4. 与蜂群验收的衔接

军团模式不替代蜂群层的验收：

| 层级 | 验收责任 | 验收方式 |
|------|---------|---------|
| L2（蜂群） | 各蜂群统帅 | **R4a** 冻结表 + 统帅实测 + **R4b** 关门 → **PASS / COND_PASS / FAIL** |
| L3（军团） | 军团统帅 | 回声校验 + 跨部门一致性 |
| 最终交付 | 军团统帅 | 汇总所有蜂群验收结果 + 军团回声校验 → 向用户报告 |

**禁止**：军团层面 PASS 覆盖蜂群层面的 FAIL；跳过蜂群验收直接在军团层面宣布完成。

## 5. 常见误读

| 误读 | 正读 |
|------|------|
| 「回声校验 = 重做一遍所有工作」 | 回声校验是**比对检查**，不是重新执行 |
| 「S 级不需要任何对抗」 | S 级不启动蜂群/军团，但统帅仍按总线纪律工作 |
| 「军团越大越好」 | 最小充分部门集；浪费资源的全编制**不优于**精准的核心编制 |
| 「降级 = 失败」 | 降级是资源约束下的合理策略，关键是保住 L2 隔离底线 |
