# 第三方 API 与集成层蜂群协议

**范围**：任何「本机服务 → 外部网关 → 模型/能力」的链路（含 OpenAI 兼容 Chat Completions、SSE、tools、多厂商 Base URL）。**不新增固定角色**，仍用 R1–R4 + 统帅。

**单一事实来源**：强制条款以 CLAUDE.md §二 为准；本文件提供 **操作清单、Agent 子代理提示模板、验收表补充列**。

---

## 1. 问题分层速查（统帅首轮归类）

| 层 | 典型信号 | 优先动作（示例） |
|----|----------|------------------|
| **传输** | `ECONNREFUSED`、TLS、5xx、超时 | 查网络、URL、密钥、厂商状态页 |
| **契约** | 200 + `choices:[]`、JSON/SSE 与请求声明不一致 | **R2** 文档 + **最小 curl**；**R3** 兼容/降级 |
| **策略/配额** | 200 + 空壳、`usage.completion_tokens===0`、无 `error` 嵌套 | 控制台用量/风控；换模型或改参数组合 |
| **解析** | 同一条 curl 有正文但本机无 | **R3** 修解析；**R4** 用**真实片段**做单测 |

**反模式**：未分层即多轮只改「提示词」或只改「JSON.parse」。

---

## 2. R2 Agent 子代理最小信息集（可复制）

- 厂商 + 端点类型（OpenAI 兼容 / 自研）。
- **实际** `model` 字符串、`stream` / `tools` / `tool_choice` 是否启用。
- **现象**：HTTP 状态、响应 `Content-Type`、**脱敏后的响应体截断**（或说明如何抓包）。
- **要交付**：官方文档条款或社区可核验结论；**一条可复制 curl**（密钥占位符）；与现象是否一致。

---

## 3. →R3 转化（硬约束示例）

- 兼容策略：如「流式失败则回退 JSON」「`tool_choice` 降级为 `auto`」。
- **观测**：开发态环境变量开关、日志截断长度、**禁止默认在生产打印全文**。
- 与用户可见错误码/文案对齐（契约层 vs 解析层分码）。

---

## 4. R4 验收补充（超越纯 mock）

**P0 建议至少包含其一**：

1. **夹具/录制**：脱敏的真实 JSON（或 SSE 多行文本）作为 Vitest/契约测试输入。
2. **可开关观测**：如 `AI_UI_DEBUG_LLM=1` 类开关在 DoD 中写明：何时开、打什么、谁负责关。
3. **手工冒烟**：对真实密钥环境的一条 **TC-xxx**（可标为「需密钥，CI skip」）。

**禁止**：将「mock 返回理想 `choices`」作为 LLM 路径 **唯一** P0 依据。

---

## 5. 统帅核验子代理（防「已实现」幻觉）

在汇流写「已解决」之前：

- [ ] `git diff` / 或等价：子代理声称的文件是否真存在修改。
- [ ] `npm test` / 约定 CI：是否绿。
- [ ] 子代理摘要与仓库 **冲突** → 以仓库为准，**重发 R3/R2 Agent 子代理**。

---

## 6. R1 可选切入点（集成向）

- 密钥、日志、提示词是否经日志外泄。
- 空响应时 UI 是否误导为「解析错误」。
- 多租户/并发下外部配额耗尽的可测行为。
