# 蜂群可验证结论与跨会话裁决落盘

**范围**：协作治理与集成假设的**可继承性**；不替代具体产品仓库内的 `README`/测试文档（产品契约仍以各仓库为准）。

## 1. 结论三级（主帅汇流强制标注）

| 等级 | 含义 | 主帅可否写进「已拍板」 |
|------|------|------------------------|
| **A 可验证** | 附 **可复核证据**：官方文档 URL+条款、终端退出码+关键输出摘要、仓库路径+行号、已编号 **TC** 的实测记录、CI 任务名 | 是 |
| **B 待核实** | 合理推断、内部一致但未对外或实测复核 | 否；须标为待决或触发 R2/实测 |
| **C 观点** | 经验、偏好、无证据链的「应该」 | 否；仅可作讨论或记入「已知假设」 |

**禁止**：把 **B/C** 当作「蜂群已验证」写进给用户的主结论而不加标签。

**叙事稿 ≠ A 级**：主帅汇流的**可读叙事**、对外脱敏摘要、或「自然语言对话导出」仅服务**人类阅读与 onboarding**，**不得**单独充当 **A 级**证据；A 级仍须路径、命令+退出码、官方条款、已执行 TC 等可复核指针（与 CLAUDE.md「留痕三档」一致）。

**身份与意图三字段**：用户**原文明示**且无矛盾时可标 **A**；统帅**推断**的须 **B（待核实）**。**假定执行者身份** + **高敏感下游用途** 时，依赖该假定的结论**默认不得**标 **A**，直至用户确认或可核验指针（见 CLAUDE.md「用户身份与意图冻结」）。

## 2. 各角色输出底线（Agent 子代理描述可引用本节）

- **R1**：边界与威胁优先；若提出「必须防 X」，建议给出 **可测负例** 或 **→R4** 条目方向，避免纯叙事。
- **R2**：事实类陈述默认 **A 级** 或明确标 **推断/待核验**；附 **来源类型**（官方文档 / 实测 / 社区）。禁止无引用时自称绝对结论。
- **R3**：方案须含 **回滚点** 与 **与现有代码/约定的冲突检查**；引用路径时给**稳定路径**而非臆测。
- **R4**：除 DoD 外，须含 **可执行** 步骤或命令占位（与环境无关处）；**P0** 优先可自动化或可脚本化；避免仅「建议多测试」。

## 3. 落盘触发（与 CLAUDE.md 一致）

- **必须落盘**：**高风险回合**（见 CLAUDE.md）；**修改本工作区 CLAUDE.md 语义**；**第三方集成**已定型兼容/降级策略（厂商+SKU+关键参数组合）。
- **建议落盘**：满编四轮且存在**跨会话会复用**的架构取舍；用户显式要求「记下来」。
- **载体**：优先 `.claude/swarm/RECENT-DECISIONS.md` 追加；大议题用 `.claude/swarm/decisions\YYYY-MM-DD-短标题.md`。
- **减负**：同一决策链**尽量一条**追加或**一个**专题文件，避免每个子代理各写一段重复简报。

## 4. 主帅向 Sub-agent 回灌证据

子代理**默认看不到**主帅终端历史。若后续 Agent 子代理依赖某次 `npm test`/curl 结果，主帅须在 **Agent 子代理正文**中粘贴 **关键输出摘要** 或 **指向** 已落盘条目路径。

**与留痕三档衔接**：回灌指针优先落在**结构化全记录**（验收表、轮次表、命令结果摘要）；**字节级镜像**仅作审计对账，须在指针中说明脱敏范围；若因体积或敏感省略片段，须在结构化记录中写 **`OMIT_REASON`**（见 CLAUDE.md 枚举与技能 `12`）。

## 5. 与 CI 的关系

**跨会话的真门禁**优先是 **绿 CI + P0 TC**；落盘记录用于**人类与下一会话 LLM**对齐假设，不替代自动化。
