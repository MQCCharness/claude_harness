# OP · 末端执行

**L3-OP（Operation）** 是军团的执行单元。每个叶 OP 负责一个具体可交付物，并自带满编蜂群小队。

---

## 职责范围

- **具体执行**：完成 MG 分配的单个可交付物（模块/页面/文档/测试套件）
- **带蜂群小队**：叶 OP 内部启动 R1–R4b 五段满编
- **质量交付**：通过蜂群全流程（R1+R2→R3→R4a→R4b）后交付

---

## 执行方式

OP 节点在阶段五须**至少一次独立 Agent 子代理**。此外，OP 作为蜂群统帅（+1），须对内部的 **R1、R2、R3、R4a、R4b 分别派发 Agent 子代理**。

---

## 叶 OP 与末端蜂群

```
OP (叶) → 作为蜂群 +1 统帅
  ├── R1 Agent (需求撕裂)
  ├── R2 Agent (技术调研)
  ├── R3 Agent (核心开发)
  ├── R4a Agent (验收清单)
  └── R4b Agent (关门核验)
```

### 强制规则

- 蜂群时序闸门必须遵守
- R1+R2 关门后方可 R3
- R4a 与 R3 可并行（黑盒契约级）
- R4b 后置于 R3 关门 + 证据回灌

---

## 产出物

- **OP-DELIVERY**：可交付物（代码/文档/配置）+ 交付说明
- **R4a 清单**：TC-B/TC-E 验收表
- **R4b 核验**：关门核验报告
- **裁决结果**：PASS / COND_PASS / FAIL

---

## 非叶 OP

非叶 OP（即 OP 下还有子 OP）不直接带蜂群，职责更接近 MG，负责协调子 OP。
