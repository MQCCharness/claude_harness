# 叶 OP 与末端蜂群

当军团已激活 + 阶段四冻结含叶 OP + 阶段五末端交付时，**强制 R1/R2/R3/R4a/R4b 五段 Agent 子代理**。

---

## 叶 OP 的蜂群结构

```
叶 OP Agent（队内 +1 统帅）
  │
  ├── R1 Agent ── 需求撕裂 → 边缘/矛盾/安全
  ├── R2 Agent ── 技术调研 → 选型/文档/CVE
  ├── R3 Agent ── 核心开发 → 代码落地
  ├── R4a Agent ─ 验收清单 → TC-B/TC-E（黑盒）
  └── R4b Agent ─ 关门核验 → 对照清单事实核对
```

---

## 强制规则

### 数量强制

R1、R2、R3、R4a、R4b 各至少 **一次** Agent 子代理。不得合并角色。

### 时序闸门

```
R1 + R2 并行
  ↓ (两者关门)
R3 ∥ R4a（R4a 黑盒，不依赖 R3 实现）
  ↓ (R3 关门 + 证据回灌)
R4b 关门核验
  ↓
队内 +1 裁决 PASS / COND_PASS / FAIL
```

### 黑盒纪律

R4a 仅基于**契约级输入**（军令状 ②③⑤、REQ、环境矩阵），**禁止**纳入 R3 实现细节。

---

## 与军团主统帅的关系

```
军团统帅（主对话 +1）
  │
  ├── EX Agent
  ├── MG Agent
  └── OP Agent（叶）→ 作为蜂群 +1
        ├── R1 Agent
        ├── R2 Agent
        ├── R3 Agent
        ├── R4a Agent
        └── R4b Agent
```

军团主统帅不直接管理叶 OP 内部的蜂群 — 由 OP 的队内 +1 负责调度。军团统帅只收 OP 的最终交付物和裁决结果。

---

## 穿透军令状

叶 OP 内部每个 R1–R4b Agent 的 prompt 必须携带：

- 军令状 **②**（冻结目标句）
- 军令状 **③**（硬约束）
- 军令状 **⑤**（成功标准）
- `[PENETRATING]` 标记
