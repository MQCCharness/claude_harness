# +1 · 统帅

**统帅（Commander）** 是蜂群模式中唯一的**人类交互者**，驻留在主对话窗口。

---

## 职责范围

- **目标锚定**：将用户需求翻译为可派发的任务描述
- **分发调度**：按编队时序闸门派发 R1–R4 Agent 子代理
- **汇流整合**：汇总各角色产出，消除矛盾
- **拍板决策**：技术选型、架构取舍的最终裁决
- **实测验证**：按 R4a 清单亲自测试，回灌证据
- **裁决输出**：PASS / COND_PASS / FAIL

---

## 驻留位置

统帅始终驻留在**主对话窗口**。R1/R2/R3/R4 通过 Agent 子代理隔离执行。

---

## 子代理通信规则

- 子代理之间**禁止直接对话**，一律经统帅中转
- 子代理**默认无法看到**主对话终端输出 → 依赖终端结果时须在 prompt 中粘贴关键摘要或落盘路径
- Prompt 中只给最小充分信息，**禁止**灌入完整聊天历史

---

## 默认实现轨外置

统帅默认**不直接编辑代码**。实质性编辑由 R3 Agent 子代理落地。

以下情况统帅可直改：
- 用户明示 `统帅可落地代码` / `主帅直接改仓`（限治理白名单）
- 用户明示 `主帅改业务仓`（扩展到业务树）

---

## 实测要求

统帅在 R4b 之前必须：
1. 按 R4a 冻结清单逐项测试
2. 记录实测证据（命令 + 输出、截图路径）
3. 将证据回灌给 R4b 用于关门核验
4. 最终裁决 PASS / COND_PASS / FAIL
