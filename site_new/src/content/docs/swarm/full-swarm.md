# 满编蜂群与写码回合

## 满编蜂群（五段强制）

**口令**：`满编蜂群` / `蜂群满编` / `五段满编` / `R1到R4b满编`

触发后强制要求：**R1、R2、R3、R4a、R4b 各至少一次 Agent 子代理**。

| 要求 | 普通蜂群 | 满编蜂群 |
|------|---------|---------|
| R1 | 写码回合触发 | 强制至少 1 次 |
| R2 | 写码回合触发 | 强制至少 1 次 |
| R3 | 强制 | 强制 |
| R4a | 强制 | 强制 |
| R4b | 强制 | 强制 |

---

## 写码回合最低触达

**触发条件**：即将或正在对本仓库做实质性代码改动。

### 强制底线

| 要求 | 说明 |
|------|------|
| R3 + R4a + R4b | 各至少 1 次 Agent 子代理 |
| 时序闸门 | R1+R2 关门 → R3；R4a∥R3；R4b 后置 |
| 统帅实测 | 按 R4a 表亲自实测后裁决 PASS/COND_PASS/FAIL |

### R4a 黑盒纪律

**仅基于契约级输入**（REQ、冻结目标、环境矩阵），**禁止**纳入 R3 实现细节。

---

## 豁免

| 口令 | 效果 |
|------|------|
| `跳过 R3R4` / `免 R3R4` | 豁免写码回合 R3+R4a+R4b |
| `不跑实测` / `豁免验收门禁` | 豁免 R4 实测裁决 |
| `免子代理` / `不要 Task` | 豁免 Agent 隔离 |
