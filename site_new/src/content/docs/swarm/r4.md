# R4a / R4b · 独立测试员

独立测试员分为两个阶段：R4a 制定验收清单（与 R3 并行），R4b 在 R3 完成后核验。

---

## R4a · 清单阶段

**核心使命**：基于契约级输入制定黑盒验收清单，不依赖 R3 实现细节。

### 黑盒纪律

- **仅基于**：REQ（需求文档）、冻结目标句、环境矩阵
- **禁止**：纳入 R3 实现细节、代码路径、内部架构假设
- **产出**：编号验收表（TC-B-xxx / TC-E-xxx）

### TC 前缀

| 前缀 | 含义 |
|------|------|
| **TC-B-** | 业务/契约类测试用例 |
| **TC-E-** | 工程/构建类测试用例 |

### 优先级

| 级别 | 含义 |
|------|------|
| **P0** | 阻断级 — 不通过则 FAIL |
| **P1** | 重要 — 不通过则 COND_PASS |

---

## R4b · 关门核验

**核心使命**：对照 R4a 冻结清单，结合 R3 实现 + 统帅实测证据，做事实核对。

### 前置条件

- R3 已完成并关门
- 统帅实测证据已回灌
- R4a 清单已冻结

### 核验内容

- 逐条对照 R4a 清单
- 标记 PASS / FAIL / BLOCKED
- 指出漏项（R4a 未覆盖但实际发现的问题）
- 确认 P0 是否全部覆盖

---

## 执行方式

R4a 和 R4b 各需至少一次 **Agent 子代理**（`subagent_type: "general-purpose"`），禁止主对话分饰。

---

## 裁定标准

| 裁定 | 条件 |
|------|------|
| **PASS** | P0 全部通过 |
| **COND_PASS** | P0 通过，P1 有未闭环 |
| **FAIL** | 任一 P0 未通过 |
